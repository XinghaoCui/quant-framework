# 如何新增一个因子

## 因子的接口

因子是一个函数：

```python
def my_factor(prices: pd.DataFrame, **kwargs) -> pd.DataFrame:
    """返回和 prices 同形状的打分 DataFrame，值越大表示越看多。"""
    ...
```

输入：宽表价格（index=date, columns=symbol）
输出：宽表因子值（同形状）

## 步骤 1：在 `src/factors/` 下加函数

```python
# src/factors/technical.py 或新建 fundamental.py
def my_factor(prices: pd.DataFrame, window: int = 60) -> pd.DataFrame:
    """
    因子思路：xxx
    数值大 = 看多。
    """
    return ...
```

## 步骤 2：在 `__init__.py` 注册

```python
# src/factors/__init__.py
from .technical import ..., my_factor
__all__ = [..., "my_factor"]
```

## 步骤 3：（可选）写一个使用此因子的策略

```python
# src/strategies/my_factor_strategy.py
from ..factors import my_factor
from .base import Strategy

class MyFactorStrategy(Strategy):
    def generate_weights(self, prices):
        score = my_factor(prices, window=self.window)
        # 按 score 横截面选股，参考"模板 2"
        ...
```

## 因子写法的几个原则

### 1. 方向统一：值大 = 看多
所有因子的输出都按"值越大越看多"约定。
如果是"低 PE 好"，写成 `1/PE` 或 `-PE`。
这样多因子合成时不用每个因子单独考虑方向。

### 2. 时间一致：用 t 时刻能知道的数据
不能用 t+1 的价格算 t 的因子。
所有 rolling/shift 都应该向"过去"看，不能向"未来"看。

### 3. NaN 处理：自然产生的 NaN 保留
前几天没数据 → NaN 没关系。
策略层会 dropna 后再排序。
不要 fillna(0)！因为 0 是有意义的因子值，会污染排序。

### 4. 横截面归一化（可选）
如果你想直接看因子值分布，可以加一个 z-score：

```python
def my_factor(prices, normalize=False):
    raw = ...
    if normalize:
        raw = raw.sub(raw.mean(axis=1), axis=0).div(raw.std(axis=1), axis=0)
    return raw
```

但策略层做排序时不需要归一化，因子原始值就够。

## 常见因子模板

### 模板 1：价格类（动量、反转、波动）

```python
def momentum_n(prices, n=252, skip=21):
    past = prices.shift(skip)
    return past / past.shift(n) - 1

def volatility_n(prices, n=60):
    return prices.pct_change().rolling(n).std() * np.sqrt(252)

def reversal_n(prices, n=21):
    return -(prices / prices.shift(n) - 1)
```

### 模板 2：技术指标类

参考 `src/factors/technical.py` 现有的 RSI、布林带等。
区别于"原始因子"在于：常用于"超买超卖"判断，可能不是单调的"值大=看多"。

### 模板 3：成交量类（需要 volume 数据）

```python
def turnover_factor(volume, market_cap):
    return volume / market_cap  # 换手率

def amihud_illiquidity(returns, volume, n=20):
    return (returns.abs() / volume).rolling(n).mean()
```

> 本框架目前数据层只有价格，没接成交量和市值。
> 要扩展，先改 `src/data/loader.py` 让它能返回更丰富的字段。

### 模板 4：基本面类（需要财报数据）

```python
def pe_factor(price, eps):
    return -price / eps  # 低 PE 好 → 负号

def roe_factor(net_income, equity):
    return net_income / equity
```

> 本框架目前不接基本面数据。
> 扩展方案：用 akshare 的 `stock_financial_abstract_em` 或专业付费源。

## 如何评估因子（独立测试）

不需要写完整策略，写一个 IC 测试函数：

```python
# 工具：测试因子的 IC
def factor_ic_test(factor: pd.DataFrame, prices: pd.DataFrame, holding=21):
    """每期算横截面 IC，返回 IC 序列、IC 均值、Rank IC、IC IR。"""
    forward_return = prices.shift(-holding) / prices - 1
    ic = factor.corrwith(forward_return, axis=1)  # 每行（每天）的相关系数
    rank_ic = factor.rank(axis=1).corrwith(forward_return.rank(axis=1), axis=1)
    return {
        "IC mean": ic.mean(),
        "IC std": ic.std(),
        "IC IR": ic.mean() / ic.std() * np.sqrt(252 / holding),
        "Rank IC mean": rank_ic.mean(),
    }
```

## 多因子合成的几种方式

### 方式 1：等权 z-score 合成
```python
score = (zscore(f1) + zscore(f2) + zscore(f3)) / 3
```

### 方式 2：IC 加权
按各因子历史 IC 加权，IC 高的权重大。

### 方式 3：IC IR 加权（最常用）
```python
weights = [ic_ir_1, ic_ir_2, ic_ir_3]
weights = weights / sum(weights)
score = sum(w * zscore(f) for w, f in zip(weights, factors))
```

### 方式 4：因子正交化
如果因子之间相关性高（如动量和盈利），可以用 Gram-Schmidt 正交化或 PCA。

## 一个完整的扩展路线建议

1. 加 1 个新因子（如 60 日波动率）
2. 写 IC 测试，看是否单调
3. 写一个用这个因子的简单策略
4. 跑回测，看与原有策略相关性
5. 如果低相关，加入多因子合成
6. 持续迭代

每周加 1-2 个，半年就有 10+ 因子的库。
这就是个像样的"个人因子库"了，简历上有亮点。
