## 写策略的几个关键约束

### 1. 不能调数据接口
策略只接受 `prices` 参数，不要在 `generate_weights` 里调 `load_prices`。
原因：可测试性、复用性。

### 2. 不能 shift 自己的信号
引擎会自动 `shift(1)`。你 shift 一次就是用了今天信号、引擎再 shift 一次就变成大前天的信号了。

### 3. 权重要归一化
确保每行 `|weights|.sum() ≤ 1`，否则相当于加杠杆。

### 4. 处理 NaN
`prices.rolling(60)` 在前 59 天会产生 NaN。你的 weights 在那些行应该是 0，不是 NaN。

### 5. 调仓频率自己控制
如果你想月度调仓，自己做 `index[::21]`。引擎对你每天的 weights 都生效。

## 常见的策略模板

### 模板 1：时序信号（单标的独立判断）

```python
def generate_weights(self, prices):
    # 对每只票独立算信号
    signal_df = ...  # +1/-1/0
    # 等权分配
    n_active = signal_df.abs().sum(axis=1).replace(0, 1)
    return signal_df.div(n_active, axis=0)
```

适合：双均线、布林带、突破。

### 模板 2：横截面排序

```python
def generate_weights(self, prices):
    factor = compute_factor(prices)
    weights = pd.DataFrame(0.0, index=prices.index, columns=prices.columns)

    rebal_dates = prices.index[::self.rebalance_days]
    for d in rebal_dates:
        scores = factor.loc[d].dropna()
        if len(scores) < 4:
            continue
        n_top = int(len(scores) * self.top_pct)
        top = scores.nlargest(n_top).index
        weights.loc[d, top] = 1.0 / n_top

    return weights.replace(0, pd.NA).ffill().fillna(0.0).infer_objects(copy=False)
```

适合：动量、价值、质量等单因子排序。

## 一个好策略的"清单"

- [ ] 思想清晰：能用一句话说出"为什么会赚钱"
- [ ] 参数有上下限：不会因为参数变就完全失效
- [ ] 在不同时段（牛、熊、震荡）都跑过
- [ ] 扣完合理成本后还赚钱
- [ ] 回撤在心理可承受范围
- [ ] 换手率合理（不要每天换 100%）
- [ ] 不依赖单一极端事件（如某一年涨 100%，其余年份持平）

## 反例：什么样的"高夏普策略"是骗自己

- 用了未来函数：信号包含未来才知道的信息
- Survivorship bias：只用现存股票（忽略退市的）
- 参数过拟合：试了 200 个组合选最好的
- 不扣交易成本：双边换手 200% 还报夏普 3
- 样本太短：只回测 1 年牛市
- 用了样本内的归一化（如全样本 z-score 而非 rolling）

写策略时随时自问这些，比追求高 Sharpe 更重要。
