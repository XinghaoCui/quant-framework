# 数据源问题排查

实盘前最常见的坑就是数据问题。

## 错误 1：yfinance "Too Many Requests"

**症状**：
```
YFRateLimitError('Too Many Requests. Rate limited. Try after a while.')
```

**原因**：Yahoo Finance 对免费用户有频率限制。短时间内拉太多次（或并发请求过多）会被限流。

**对策**：
1. **等几分钟再试**（限流是临时的）
2. **减少并发**：把 `yfinance.download(threads=True)` 改成 `threads=False`
3. **加缓存**：本框架已有 pickle 缓存，第二次跑不会再请求
4. **分批拉**：一次只拉 5-10 只票，间隔 sleep 30 秒
5. **换数据源**：用 Alpha Vantage（免费 API key）、Polygon、IEX Cloud

代码修改示例（在 loader.py 里）：

```python
import time
```

## 错误 2：akshare 代理错误

**症状**：
```
ProxyError('Unable to connect to proxy', RemoteDisconnected(...))
```

**原因**：akshare 调用国内接口（东方财富、新浪、雪球等），
但你装了**全局代理**（如 Clash、V2Ray、Shadowsocks），
代理软件试图把国内流量也走 VPN，被国内服务器拒绝。

**对策**：
1. **关闭代理**：跑 A 股数据前关闭 Clash/V2Ray
2. **配置代理规则**：让国内域名直连：
   - Clash 配置里加 `DOMAIN-SUFFIX,eastmoney.com,DIRECT`
   - 同样为 sina.com.cn、xueqiu.com、akshare 用到的接口
3. **绕过代理的环境变量**：
   ```bash
   export NO_PROXY="eastmoney.com,sina.com.cn,xueqiu.com"
   ```
4. **用其他 A 股数据源**：tushare（需注册）、聚宽（需注册）

## 错误 3：数据缺失或时间不对

**症状**：拉到的数据少了某些日期，或最新日期是几个月前。

**原因**：
- yfinance 有时返回不完整数据（断断续续 NaN）
- 上市新股、退市旧股
- 节假日不同（A 股有春节连休）

**对策**：
1. 用 `prices.dropna(how='all')` 删掉全为 NaN 的行
2. 单只票缺数据：`prices.ffill(limit=5)` 补最多 5 天
3. 长期缺数据的票：从样本剔除

## 错误 4：复权问题

**症状**：某天股价"跌"了 50%，但实际是除权除息。

**原因**：用了不复权数据。

**对策**：
- A 股：用后复权（`adjust='hfq'`），价格连续可比
- 美股：yfinance `auto_adjust=True` 默认后复权
- 本框架默认 `adjust='post'`，已处理

## 错误 5：Survivorship Bias（幸存者偏差）

**症状**：回测夏普 2.5，实盘 0.5。

**原因**：你回测时用的是"今天还存在的股票"——
那些已经退市的（破产、被收购）你拉不到，
但在历史上它们是真实存在过、可能亏过钱的。

**对策**：
- 用专业数据源（CRSP、Wind、CSMAR），它们保留历史成分股
- 用 ETF 的成分股历史（如 SPY 成分股按时间点）
- 接受偏差，但回测结论保守化看待

本框架因为数据来源有限，**未规避此偏差**。学术研究中必须规避。

## 错误 6：时区/日期不一致

**症状**：A 股和美股数据合并后，索引对不上。

**原因**：
- 美股 NYSE 时区是 ET，A 股是 CST
- 不同数据源可能返回不同时区
- 节假日不同（5/1、10/1、感恩节、圣诞节）

**对策**：
- 所有时间戳统一去时区：`df.index.tz_localize(None)`
- 合并前 reindex 到共同日期：`df.reindex(common_dates)`

本框架的 `loader.py` 已经统一去时区。

## 错误 7：A 股代码格式混乱

**症状**：你输入 "600519"，但下游接口要 "sh600519" 或 "600519.SH"。

**对策**：
本框架的 akshare 接口接受 6 位纯数字（"600519"）。
如果你写了 Wind 风格 "600519.SH"，框架会自动剥掉 ".SH"。

A 股代码规则（速查）：
| 前缀 | 板块 | 交易所 |
|---|---|---|
| 60 | 主板 | 上交所 |
| 68 | 科创板 | 上交所 |
| 00 | 主板 | 深交所 |
| 30 | 创业板 | 深交所 |
| 8 / 4 | 北交所 | 北交所 |

## 数据完整性自查脚本

跑数据后建议自查：

```python
prices = load_prices(["AAPL", "MSFT"], "2020-01-01", "2024-12-31", market="us")

print("形状:", prices.shape)
print("日期范围:", prices.index.min(), "→", prices.index.max())
print("缺失值:\n", prices.isna().sum())
print("零值/负值:", (prices <= 0).sum().sum())
print("价格分布:\n", prices.describe())
```

输出里关注：
- 缺失值是否过多
- 是否有 0 或负价（数据错误）
- 日期跨度是否合理（节假日除外不应有大段跳空）

## 数据源对比

| 数据源 | 市场 | 免费 | 优点 | 缺点 |
|---|---|---|---|---|
| yfinance | 全球 | 是 | 简单，全市场 | 限流、数据偶有错 |
| akshare | A 股+全球 | 是 | 国内数据全 | 接口偶尔变 |
| tushare | A 股 | 部分免费 | 数据规范 | 高级数据要积分 |
| Alpha Vantage | 美股 | 是 (key 限频) | 稳定 | 单日请求数有限 |
| Polygon | 美股 | 付费 | 高频、tick | 贵 |
| Wind | 全球 | 付费 | 行业金标准 | 极贵，多用机构 |
| 聚宽/优矿 | A 股 | 付费 | 因子库全 | 仅平台内用 |

**学习阶段**：yfinance + akshare 足够。
**实盘准备**：要至少一个付费可靠源。
