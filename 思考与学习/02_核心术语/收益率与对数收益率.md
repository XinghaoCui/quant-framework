# 简单收益率 vs 对数收益率

这是量化金融第一个会让人犯错的细节。

## 定义

设资产价格从 $P_{t-1}$ 变到 $P_t$：

- **简单收益率（Simple return）**: $r_t = P_t / P_{t-1} - 1$
- **对数收益率（Log return）**: $\ell_t = \ln(P_t / P_{t-1}) = \ln(1 + r_t)$

例：100 → 110
- 简单收益 = 10/100 = 10.00%
- 对数收益 = ln(1.10) ≈ 9.53%

## 两者什么时候差很多？

收益率小时两者差不多（一阶泰勒近似 $\ln(1+x) \approx x$）。

收益率大时差很多：
- +50% 简单收益 → +40.5% 对数收益
- -50% 简单收益 → -69.3% 对数收益（注意是 -∞ 极限）

## 各自的"代数性质"

### 简单收益：可以乘，不能加

```
两期累计 = (1 + r_1)(1 + r_2) - 1
```

不能直接 `r_1 + r_2`。

例：第一天 +10%，第二天 -10%：
- 错误："0%"
- 正确：1.1 × 0.9 - 1 = -1%

### 对数收益：可以加，不能乘

```
两期累计对数收益 = ℓ_1 + ℓ_2
```

因为 $\ln(P_2/P_0) = \ln(P_2/P_1) + \ln(P_1/P_0)$。

这是对数收益最大的优势：时间可加。

## 什么时候用哪个？

| 场景 | 用哪个 | 为什么 |
|---|---|---|
| 累计净值曲线 | 简单收益 | `(1+r).cumprod()` 直接给净值 |
| 计算波动率/夏普 | 对数收益 | 统计上近似正态，且与时间长度成线性关系 |
| 单期描述（"今天涨了 X%"） | 简单收益 | 更直觉 |
| 多期模型（GBM、波动率年化） | 对数收益 | 因为 $\ell_t \sim N$ 是模型假设 |

实际工程中：
- 你的回测净值用 `(1 + r).cumprod()` 这种简单收益的累乘
- 你的统计指标（年化波动 × √252）严格说应该基于对数收益
- 但在日级别上两者差异极小，工程上常常混用，没人在意

## 年化波动率为什么是 ×√252

前提：假设日（对数）收益率是独立同分布 (iid)。

设日收益方差为 $\sigma_d^2$。
N 个独立日的收益和的方差 = $N \sigma_d^2$。

所以：
- 一年的方差 = $252 \sigma_d^2$
- 一年的标准差 = $\sqrt{252} \sigma_d$

这就是 √252 的来源。

注意：这个推导对对数收益严格成立（因为对数收益可加），
对简单收益只是近似（在小波动下还行）。

## A 股 252 还是 242 天？

学术界惯用 252（接近美股年交易日数）。
A 股精确是约 242 天/年。
两者夏普差异约 √(252/242) ≈ 1.02，可以忽略。
保持一致比"精确"更重要，你只要做横向对比时用同一个常数就行。

## 代码里要注意的点

```python
import numpy as np
import pandas as pd

prices = pd.Series([100, 110, 99, 105])

# 简单收益
simple_ret = prices.pct_change()
# 累计净值
net = (1 + simple_ret).cumprod()

# 对数收益
log_ret = np.log(prices / prices.shift(1))
# 累计对数收益
cum_log = log_ret.cumsum()
# 换算回净值
net_from_log = np.exp(cum_log)
```

两条 `net` 数值上完全一致，只是路径不同。

## 一个常见错误

```python
# 错误：把对数收益直接当百分比展示给用户
ret = np.log(price[1:] / price[:-1])
print(f"昨日涨幅 {ret[-1]:.2%}")
```

如果价格从 100 涨到 120，对数收益 = 18.23%，而用户期待的"涨幅"是 20%。

**对外展示一律用简单收益；内部建模可以用对数收益。**
