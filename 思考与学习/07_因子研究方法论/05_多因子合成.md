# 多因子合成

单因子 IC 通常 0.03~0.08, 信号弱、容易失效。把几个**相关性低**的因子合成综合分, 是业界标准做法:

$$
\text{综合分} = w_1 \cdot \tilde{f}_1 + w_2 \cdot \tilde{f}_2 + \cdots + w_k \cdot \tilde{f}_k
$$

其中 $\tilde{f}_i$ 是标准化后的因子, $w_i$ 是权重。
本篇讲: 为什么要合成、怎么标准化、怎么定权重。

## 一、为什么合成能赢

数学上, 几个**独立** (或低相关) 信号的合成, 信号强度 ∝ √k:

```
单因子: IC = 0.05, IR = 0.4
3 个独立因子合成: IC ≈ 0.05 (差不多), IR ≈ 0.4 × √3 ≈ 0.69 (高 70%)
5 个独立因子合成: IR ≈ 0.4 × √5 ≈ 0.89 (高 120%)
```

**直觉**: 单因子像一个有偏见的裁判, 几个独立裁判投票更准。

但有个**致命前提**: 因子要**低相关**。如果两个因子高度相关 (corr=0.9), 合成等于没合成。
所以多因子研究的第一步是**因子相关性矩阵**:

```python
import pandas as pd

factor_corr = pd.DataFrame({
    "momentum":  momentum_series,
    "reversal":  reversal_series,
    "lowvol":    lowvol_series,
}).corr()
```

经验法则: 因子两两相关 |corr| < 0.5 比较理想, > 0.7 就要考虑去掉一个。

## 二、必须先标准化 (Standardization)

错误做法 (直接相加):

```python
score = momentum + reversal + lowvol   # 错!
```

为什么错? 三个因子的**量级完全不同**:
- 动量因子: 可能在 -50% ~ +200% 之间
- 反转因子: 可能在 -30% ~ +30% 之间
- 低波因子: 可能在 0.1 ~ 1.0 之间

不标准化, 动量因子的范围最大, 就独占了综合分的话语权 — 其他因子等于没加。

### 标准化方法 1: 截面 zscore (推荐)

每天独立做:
$$
\tilde{f}_i(t) = \frac{f_i(t) - \bar{f}(t)}{\sigma_f(t)}
$$

- 减去**当天截面均值**, 除以**当天截面标准差**
- 标准化后每个因子在每一天截面上均值 0、标准差 1
- 量级统一, 可以直接相加

**为什么必须按日做, 不能用整段时间的均值/标准差?**
后者会引入未来信息: "今天的 zscore"用到了"未来才知道的整段均值", 是典型的未来函数。

### 标准化方法 2: 截面 rank
更稳健 (抗异常值):
$$
\tilde{f}_i(t) = \frac{\mathrm{rank}(f_i(t))}{N(t)} - 0.5
$$

每天把因子值转成 [-0.5, +0.5] 的等距 rank。对极端值不敏感, 大资金风控偏好。

本框架默认用 zscore (实现见 `src/strategies/multi_factor.py:cross_section_zscore`)。

## 三、权重怎么定: 等权 vs IC 加权

### 方法 1: 等权 (equal-weighted)

$$
w_i = \frac{1}{k}
$$

最简单, 不需要参数。出乎意料地难打: 等权是对所有 IC 估计的**极强收缩** (Bayesian shrinkage), 反而避免噪声。

> 学术界很多论文显示: 等权在样本外通常打不过, 但也输不太多。
> 复杂权重模型样本内夏普惊人, 样本外经常不如等权。

### 方法 2: IC 加权 (IC-weighted)

$$
w_i = \frac{\overline{IC_i}}{\sum_j |\overline{IC_j}|}
$$

按每个因子**最近一段时间的 IC** 加权:
- IC 高的因子权重高
- IC 为负的因子自动**反向使用** (权重为负)
- IC 接近 0 的因子权重接近 0 (被自动剔除)

**关键工程细节**: 滚动 IC 必须用**只能"看到过去"的数据**算, 不能用未来 IC 决定今天的权重。
本框架的实现:

```python
# src/strategies/multi_factor.py (节选)
rolling_ic = ic_df.rolling(self.ic_lookback, min_periods=10).mean().shift(self.ic_horizon)
```

`shift(ic_horizon)` 这一步至关重要: t-horizon 天的 IC 用到了 t 天的真实收益, 已经发生; 不 shift 就是用未来信息。

### 方法 3: ML 学权重
线性回归 / Lasso / Ridge / 神经网络都能学多因子权重。这就是 06 章 ML 模块在做的事。
ML 的优势: 学非线性、交互项; 劣势: 极易过拟合。

## 四、选哪种?

实用决策树:
- **入门 / 因子很少 (k<5) / 简单稳健**: equal
- **因子较多 (k=5~20) / 有把握做防泄露 rolling IC**: ic_weighted
- **大规模因子库 (k=20+) / 想发现交互**: 上 ML (跳到 06 章)

> 强烈建议先 equal 打底, 然后 ic_weighted 对比, 看是否真的提升。
> 经常 ic_weighted 跑出来不如 equal, 就别用复杂方法。


## 六、常见陷阱

### 1. 因子方向不一致
要求所有因子都"值越大越看多"。如果某个因子是反向的 (如波动率), 传入前**自己取负**。
框架不会自动判断, 这是研究员的责任。

### 2. NaN 处理
不同因子的 NaN 分布不同 (动量需要 252 天 lookback, 反转只要 21 天), 早期日子可能某些因子全是 NaN。
本框架对每个因子独立 zscore 后求和 (NaN 自动跳过), 实战可行。

### 3. 过去 IC 不代表未来 IC
ic_weighted 隐含假设"最近 60 天 IC 高的因子下个月还会高", 这在因子拥挤、régime 切换时会失效。
**始终保持 equal 作为参照**, 一旦 ic_weighted 明显输给 equal, 说明 IC 估计在追噪声。

### 4. 标准化要按日做
绝对不要用 `factor.zscore()` (全局), 必须 `cross_section_zscore` (按行)。
全局 zscore = 用未来均值标准化 = 未来函数。

## 七、合成 vs 选 top-k 因子

另一种思路: 不合成综合分, 直接"用最近表现最好的 k 个因子各自选 top"。
比如有 10 个因子, 选近期 IC 最高的 3 个, 每个独立选 top, 然后取并集。

优点: 避免合成的 model risk (你选了错的合成方法)。
缺点: 仓位重叠管理复杂, 没有"信号叠加"效应。

这是个值得探索的方向, 本框架暂不实现, 留作扩展。

## 八、小结

- 合成的本质: 几个低相关弱信号 → √k 倍的强信号
- 前提: 因子必须低相关 (|corr| < 0.5)
- 必须先**截面**标准化 (zscore 或 rank), 不能全局标准化
- 权重三种: equal (难打)、ic_weighted (滚动 IC, 当心防泄露)、ML
- 入门: 先 equal 再 ic_weighted, 永远跟基准对比
- 框架接口: `MultiFactorStrategy(factors, combine="equal" | "ic_weighted")`

最后一篇把这一章串成完整的"因子研究流程"。
