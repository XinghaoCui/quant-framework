# 完整因子研究流程

本章前 5 篇分别讲了 IC、IR、分组、衰减、合成。
最后一篇把它们串成一个**标准研究流程**, 让你拿到任何新因子都能照搬。

这个流程是商业量化研究员的日常工作模板, 也是面试常问的"你怎么研究一个因子"。

## 一、流程总览

```
0. 提出因子假设 (经济机制)
   ↓
1. 实现因子, 检查数据问题
   ↓
2. 单因子 IC / IR / t 检验           ← 有用吗 + 稳不稳
   ↓
3. 分组回测 / 单调性检验              ← 真的高低分明吗
   ↓
4. IC 衰减分析                       ← 信号能维持多久
   ↓
5. 与已有因子的相关性                 ← 是不是已有因子的变种
   ↓
6. 加入多因子组合, 看综合贡献          ← 真正的最终价值
   ↓
7. 决定上线还是丢弃
```

下面逐步详解, 配套 `src/factors/evaluation.py` 的 API。

## 二、第 0 步: 经济机制假设

**先想清楚因子为什么应该有效, 再去算。**

例:
- 动量因子: "投资者跟风, 涨势会延续" (行为金融学)
- 价值因子: "便宜的好公司被低估, 未来均值回归" (DCF 估值偏差)
- 低波动: "高波动股票更受散户追捧导致定价过高" (风险溢价倒挂)

**没有机制假设的因子叫"数据挖掘"**, 样本内夏普再高也很可能是 spurious correlation。
研究员们对"找到漂亮 IC 但讲不出故事"的因子保持高度警惕。

## 三、第 1 步: 实现 + 数据健康检查

实现后第一件事不是算 IC, 是看因子值本身:

```python
factor.describe()                  # 范围、均值、std
factor.isna().sum() / len(factor)  # NaN 比例
factor.quantile([0.01, 0.99])     # 1%/99% 分位, 看异常值
```

典型问题:
- NaN 太多 (>30%): 因子计算逻辑有问题或数据缺失严重
- 1%/99% 分位差异巨大: 长尾分布, IC 算 Pearson 会被异常值毒杀
- 整段时间因子值不变: 数据更新频率有问题 (如季报因子)

**这一步发现的问题后面再修就麻烦了, 一定要做。**

## 四、第 2 步: 单因子 IC 检验

```python
from src.factors.evaluation import compute_ic, ic_summary, forward_returns

for h in [5, 20, 60]:
    fwd = forward_returns(prices, h)
    ic = compute_ic(factor, fwd, method="spearman")
    stats = ic_summary(ic)
    print(f"horizon={h}: IC={stats['IC均值']:.4f}, IR={stats['IR']:.3f}, t={stats['t统计']:.2f}")
```

判读 (Rank IC):
| 指标 | 通过线 |
|---|---|
| |IC 均值| | > 0.02 (理想 > 0.04) |
| IR | > 0.3 (理想 > 0.5) |
| |t 统计| | > 2 (5% 显著) |
| IC>0 占比 | > 55% |
| 跨多个 horizon | 至少 2 个 horizon 通过 |

**任一项不达标都先停下, 排查是否数据问题, 而不是直接下一步。**

## 五、第 3 步: 分组回测

```python
from src.factors.evaluation import quantile_backtest

result = quantile_backtest(factor, prices, n_quantiles=5, horizon=20)
print(result.summary)
```

要看的:
- **单调性**: Q1 < Q2 < ... < Q5 是不是大致单调
- **多空价差年化夏普**: > 1 算不错, > 2 顶级
- **多头驱动 vs 空头驱动**: 看 (Q5 - 中位) 和 (中位 - Q1) 哪个大
- **极端组失真**: Q1 / Q5 是不是只有几只票, 看 grouped 数据规模

**A 股做不了空, 这里重点看多头驱动是否够强**, 即 Q5 是否显著跑赢中位组。

## 六、第 4 步: IC 衰减分析

```python
from src.factors.evaluation import ic_decay

decay = ic_decay(factor, prices, horizons=[1, 3, 5, 10, 20, 40, 60, 120])
print(decay)
```

得出结论:
- IC 在哪个 horizon 达峰? → 决定**调仓周期**
- IC 累积型 (上升) 还是短促型 (下降)? → 决定**因子分类**
- 多 horizon 都正还是只有某段正? → 是不是个真信号

> 一个好因子在多个 horizon 上都应该有正 IC, 而不是"只在 h=5 一个点有"。
> 后者大概率是 horizon-cherry-picking 的假阳性。

## 七、第 5 步: 与已有因子相关性

把新因子加入因子库**前**, 必须和已有因子做相关性矩阵:

```python
existing = {
    "momentum": existing_mom_ic_series,
    "reversal": existing_rev_ic_series,
    "lowvol":   existing_lowvol_ic_series,
}
new_ic = compute_ic(new_factor, fwd)

corr = pd.DataFrame({**existing, "new_factor": new_ic}).corr()
print(corr)
```

判断标准:
- |corr| > 0.7: **基本是已有因子的变种**, 加入价值很低, 考虑直接替换或丢弃
- 0.4 < |corr| < 0.7: 有重叠但有独立信号, 谨慎加入, 看下一步
- |corr| < 0.4: **独立信号**, 强烈推荐加入

> 同样的 IC, 跟现有因子相关性 0.2 的比相关性 0.8 的价值高 5~10 倍。
> 一个有独立信号的弱因子, 比一个跟现有因子高度重叠的强因子更值钱。

## 八、第 6 步: 在多因子组合中的边际贡献

最后一步: 实际把因子加入组合, 看综合表现是否提升:

```python
from src.strategies import MultiFactorStrategy
from src.engine import Backtester

bt = Backtester()

# 基线: 现有因子组合
factors_old = {"momentum": mom, "reversal": rev, "lowvol": lv}
strat_old = MultiFactorStrategy(factors_old, combine="equal")
result_old = bt.run(prices, strat_old.generate_weights(prices))

# 加入新因子
factors_new = {**factors_old, "new_factor": new_factor}
strat_new = MultiFactorStrategy(factors_new, combine="equal")
result_new = bt.run(prices, strat_new.generate_weights(prices))

# 对比
print("旧组合:", result_old.metrics["夏普比率"])
print("加新因子:", result_new.metrics["夏普比率"])
```

判断:
- 新组合夏普明显升 → 因子有边际贡献, 上线
- 夏普持平 → 因子被现有因子覆盖, 不上线
- 夏普下降 → 因子可能反向有效, 取负号再试, 或丢弃

**这一步的对比比第 2~5 步任何一步都重要**, 因为它直接回答"这个因子真的让组合更好了吗"。

## 九、第 7 步: 决定上线还是丢弃

汇总前面 6 步的证据, 做最终决策:

| 维度 | 通过条件 |
|---|---|
| 经济机制 | 讲得清, 不是凭空挖出来的 |
| IC | |IC| > 0.02, IR > 0.3, t > 2 |
| 单调性 | Q1~Q5 大致单调, 多空价差年化夏普 > 0.5 |
| 衰减 | IC 在多个 horizon 都正, 不是只在一点 |
| 独立性 | 与现有因子 |corr| < 0.5 |
| 边际贡献 | 加入组合后综合夏普明显提升 |

**6 项全过才能上线**。任何一项不过都先停下, 想清楚再下一步。

不要因为夏普高就草率上线 (回测过拟合最常见的坑);
也不要因为夏普暂时一般就丢弃 (可能是低相关的好补充因子)。

## 十、`factor_report` 一行跑完前 4 步

为了方便, 框架提供综合报告函数, 一行打完 IC + 分组 + 衰减:

```python
from src.factors import factor_report

report = factor_report(
    factor, prices,
    name="my_factor",
    horizon=5,
    n_quantiles=5,
    decay_horizons=[1, 5, 10, 20, 40, 60],
)
print(report.to_text())
```

输出包含:
- 主 horizon 下的 IC 摘要 (IC均值/IR/t/胜率/样本)
- IC 衰减表 (多 horizon)
- 分组回测摘要 (Q1~Qn + 多空)

第 5~7 步 (相关性、组合贡献、上线决策) 要研究员手动判断, 框架不替你做。

## 十一、研究员的日常

掌握这套流程后, 你的工作流大致变成:

```
周一: 看新论文, 想出因子假设, 实现; 跑 factor_report
周二: 数据健康检查、跨 horizon IC 复测、相关性矩阵
周三: 决定加入组合; 跑多因子回测对比
周四: 写研究报告, 同事评审
周五: (通过) 上线监控 / (不通过) 归档
```

这就是私募量化研究员每周在做的事。
