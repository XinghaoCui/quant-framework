# 机器学习在量化中的定位

开篇先回答三个问题：ML 是什么？在量化里能干什么？不能干什么？

## 一、机器学习到底是什么

一句话：**从数据里自动找规律，用规律做预测。**

传统编程：人写规则，机器执行。
```
规则：if RSI < 30: 买入
```

机器学习：人给数据（输入+答案），机器自己找规则。
```
给它 10 万条 (因子值 → 未来涨跌)，它自己学出"什么样的因子组合更可能涨"
```

在量化里，"找规律"具体就是：
> 找到一个函数 f，输入股票当前的因子值 X，输出对未来收益 y 的预测。

$$
\hat{y} = f(X)
$$

ML 就是用算法（线性回归、树、神经网络）从历史数据里把这个 f 估计出来。

## 二、ML 在量化里能发挥什么作用

### 1. 多因子合成（最主流的用法）

传统多因子：人工设定权重。
```
score = 0.3*动量 + 0.3*价值 + 0.4*质量
```

ML 多因子：让模型自己学权重和非线性组合。
```
score = f(动量, 价值, 质量, 波动率, ...)  # f 由数据学出
```

优势：
- 能发现因子间的**非线性关系**和**交互作用**（如"高动量+低波动"特别好，但单独看都一般）
- 能处理几十上百个因子，人工调不过来
- 客观，不依赖主观经验

**这是本模块的核心，`src/ml/` 实现的就是这个。**

### 2. 收益/方向预测
- 回归：预测未来 N 天的收益率
- 分类：预测涨/跌

### 3. 其他场景
- **风险建模**：预测波动率、相关性矩阵
- **另类数据挖掘**：从新闻文本、卫星图像提取信号（NLP/CV）
- **执行优化**：预测最优下单时机、拆单策略
- **异常检测**：识别市场状态切换、闪崩

## 三、ML 在量化里**不能**做什么（极其重要）

### 1. ML 不是印钞机

> 金融数据的信噪比极低。预测收益的 R² 通常只有 **0.001~0.01**。

对比：
- 图像识别准确率 99%+
- 股票方向预测能到 **52%** 就是很好的策略了

为什么这么低？因为市场接近有效——如果规律明显，早就被人套利掉了。
**ML 在量化里是"沙里淘金"，不是"点石成金"。**

### 2. ML 不能凭空创造 alpha

模型只能放大特征里**已有**的信号。
如果你的因子本身没用，再强的模型也学不出东西。
> **特征工程 >> 模型选择。** 好特征配烂模型 > 烂特征配好模型。

### 3. 复杂模型 ≠ 更赚钱

这是本模块最想让你记住的一点。

看我们 `examples/12` 跑出的真实结果（模拟数据）：

| 策略 | 夏普比率 |
|---|---|
| GBDT 机器学习策略 | 0.56 |
| 等权买入持有（基准） | **1.36** |

**ML 策略输给了"啥也不想，等权买入"！**

原因：
- 模拟数据信号是线性的，复杂模型反而过拟合噪声
- ML 策略只选 30% 股票，集中持仓波动更大
- 简单基准分散充分，反而稳

教训：
> 永远先用**最简单的基准**（买入持有、等权、单因子）打底。
> 如果你的花哨 ML 模型打不过买入持有，它就没有价值。

### 4. ML 极易过拟合到"假规律"

金融数据少、噪声大，模型很容易"背下"历史噪声当规律。
样本内夏普 3.0、样本外 0.2 是家常便饭。
所以**防过拟合、严格的样本外验证**比模型本身重要得多。
详见 [03_金融数据的坑](./03_金融数据的坑.md)。

## 四、ML 量化 vs 传统量化

| 维度 | 传统因子量化 | ML 量化 |
|---|---|---|
| 规则来源 | 人工设定 | 数据学习 |
| 可解释性 | 强 | 弱（尤其深度学习） |
| 因子数量 | 几个到几十个 | 几十到几百 |
| 非线性 | 难处理 | 擅长 |
| 过拟合风险 | 较低 | **高** |
| 数据需求 | 中 | 大 |
| 上手难度 | 低 | 高 |

**没有谁取代谁**。业界主流是"传统因子做特征 + ML 做合成"，两者结合。

## 五、本模块的学习路径

```
00 定位（本篇）—— ML 是什么、能/不能做什么
   ↓
01 监督学习基础 —— 回归/分类、训练/测试、金融数据特殊性
   ↓
02 特征工程与标签 —— 把因子变成 X，把未来收益变成 y
   ↓
03 金融数据的坑 —— 数据泄露、过拟合、时序验证【最重要】
   ↓
04 线性模型 —— 从最简单的开始
   ↓
05 树模型 —— 随机森林、GBDT
   ↓
06 模型评估 —— IC、分组回测（量化的评估方式）
   ↓
07 完整工作流与进阶 —— 端到端、滚动重训、深度学习展望
```

配套代码在 `src/ml/`，示例在 `examples/10~12`。
**边读边跑边改参数**，是理解 ML 最快的方式。

## 六、给你（量化求职）的建议

面试里 ML 量化的加分项不是"我会调 XGBoost"，而是：
- 知道**为什么金融 ML 这么难**（信噪比、过拟合、非平稳）
- 知道**怎么做正确的验证**（时序 CV、purge、embargo）
- 知道**怎么评估**（IC/ICIR/分组，而不是准确率）
- 有**怀疑精神**（先打基准、警惕高夏普）

这些"判断力"比模型技巧值钱得多。本模块每一篇都在强调这些。
