# 监督学习基础

ML 分三大类，量化里 95% 用的是**监督学习**，本篇讲透它。

## 三类机器学习

| 类型 | 定义 | 量化例子 |
|---|---|---|
| **监督学习** | 给"输入+正确答案"，学输入→答案的映射 | 用因子预测未来收益 |
| 无监督学习 | 只给输入，自己找结构 | 股票聚类、降维(PCA) |
| 强化学习 | 通过试错+奖励学习决策 | 最优执行、动态调仓（前沿，难落地） |

## 监督学习的两个基本要素

### 特征 X（Features / 自变量）
描述一个样本的所有信息。在量化里就是**因子**：
```
X = [动量, 波动率, RSI, 价格相对均线位置, ...]
```

### 标签 y（Label / 因变量 / 目标）
我们想预测的东西：
```
y = 未来 5 天的收益率
```

模型的任务：学一个函数 $f$，使 $f(X) \approx y$。

## 一个"样本"是什么

这是量化 ML 最容易绕晕的地方。

在图像识别里，一个样本 = 一张图。
在量化里，**一个样本 = (一只股票, 一个时间点)**。

举例，我们有 30 只股票、2600 个交易日：
- 理论样本数 = 30 × 2600 = 78000 个
- 每个样本：某只票在某天的因子值（X）+ 它之后 5 天的收益（y）

看 `examples/10` 的真实输出：
```
特征矩阵 X：(69930, 10)   ← 69930 个样本，每个 10 个特征
标签 y：(69930,)          ← 对应 69930 个答案
```

这就是把"宽表价格"变成"长表样本"的结果，代码在 `src/ml/features.py:build_features`。

## 回归 vs 分类

监督学习按标签类型分两种：

### 回归（Regression）：预测连续值
```
y = 未来收益率 = +2.3% / -1.5% / ...
```
模型：线性回归、Ridge、随机森林回归、GBDT。

### 分类（Classification）：预测离散类别
```
y = 涨(1) / 跌(0)
```
模型：逻辑回归、随机森林分类。

### 量化里用哪个？

各有优劣：
- **回归**：信息更全（不只是方向，还有幅度），但对极端值敏感
- **分类**：更稳健（只管方向），但丢了幅度信息

实务中常见做法：
- 预测收益（回归）后，**按预测值排序**选股 → 这时回归的绝对值不重要，**排序**才重要
- 所以很多人用回归训练，但用 IC（排序相关性）评估

本框架 `MLStrategy` 默认用回归，然后按预测值排序选股。

## 训练集 / 验证集 / 测试集

ML 的铁律：**评估模型必须用它没见过的数据。**

| 数据集 | 用途 |
|---|---|
| 训练集 (Train) | 模型从这里学规律（fit） |
| 验证集 (Validation) | 调超参数、选模型（防止在测试集上偷看） |
| 测试集 (Test) | 最终评估，只用一次 |

典型划分 6:2:2 或 7:3。

### 为什么必须分开？

如果用训练数据评估，模型可以"死记硬背"得满分，但遇到新数据就崩。
这叫**过拟合**——记住了噪声，没学到规律。
分开测试，才能知道模型是真的"学会"还是"背答案"。

## 金融数据的特殊性（划重点）

普通 ML 的套路在金融上会**翻车**，因为金融数据有三个特殊性：

### 1. 时序性 → 不能随机划分
普通 ML 随机打乱划分 (shuffle)。
金融数据**绝对不能**——会把未来信息泄露进训练集。
必须**按时间切**：训练集在前，测试集在后。
详见 [03_金融数据的坑](./03_金融数据的坑.md)。

### 2. 低信噪比 → 别期待高准确率
图像识别 99%，股票方向 52% 就很棒。
评估指标也不一样：用 IC 而非准确率。

### 3. 非平稳 → 规律会失效
图像里"猫"永远是猫。
但市场规律会变（因子衰减、régime 切换）。
所以需要**滚动重训**，模型不能一劳永逸。

## 一个最小的监督学习流程（对照代码）

```python
from src.ml import make_dataset, time_series_split, ModelWrapper

# 1. 造数据集 (X, y)
X, y = make_dataset(prices, horizon=5, kind="return")

# 2. 按时间切（不是随机！）
X_tr, y_tr, X_te, y_te = time_series_split(X, y, test_size=0.3, embargo_days=5)

# 3. 训练
model = ModelWrapper("ridge", task="regression")
model.fit(X_tr, y_tr)

# 4. 预测
pred = model.predict(X_te)

# 5. 评估（量化方式：IC）
from src.ml import information_coefficient
print(information_coefficient(pred, y_te))
```

这就是 `examples/10` 的核心。跑一遍，你就走完了完整的监督学习闭环。

## 小结

- 监督学习 = 学 X → y 的映射
- 量化里：X = 因子，y = 未来收益，样本 = (股票, 时间点)
- 回归预测幅度，分类预测方向，但量化常按**排序**用
- 必须分训练/测试，且金融数据**按时间切，不能随机**
- 别期待高准确率，金融是低信噪比游戏
