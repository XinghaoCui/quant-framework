## 二、训练一次 vs 滚动重训（Walk-Forward）

### 本框架现状：训练一次
`MLStrategy` 用前 50% 数据训练一次，后 50% 全部用这个模型预测。

优点：简单、快、易理解。
缺点：忽略了市场非平稳。2015 年学的规律，2024 年可能失效。

### 进阶：滚动重训（Walk-Forward）

```
[训练 2015-2018] → 预测 2019
        [训练 2015-2019] → 预测 2020
                [训练 2015-2020] → 预测 2021
                        ...
```

每年（或每季度）用截至当时的数据重训，再预测下一段。
这才贴近实盘：模型持续吸收新数据，适应市场变化。

实现思路（扩展方向）：
```python
def walk_forward_predict(prices, retrain_freq=252, ...):
    predictions = []
    for split_date in rolling_dates(prices, step=retrain_freq):
        X_tr, y_tr = data_before(split_date)
        X_te = data_after(split_date, length=retrain_freq)
        model = ModelWrapper("gbdt").fit(X_tr, y_tr)
        predictions.append(model.predict(X_te))
    return pd.concat(predictions)
```

这是把 `MLStrategy.fit_predict` 从"一次"改成"循环"，是一个很好的练手扩展。

## 三、交叉验证选超参数

不要在测试集上调参（会污染测试集）。用 `purged_kfold` 在训练集内部交叉验证：

```python
from src.ml import purged_kfold, ModelWrapper, information_coefficient
import numpy as np

best_alpha, best_ic = None, -1
for alpha in [0.1, 1.0, 10.0]:
    ics = []
    for tr_idx, val_idx in purged_kfold(X_train, n_splits=5, embargo_days=5):
        m = ModelWrapper("ridge", alpha=alpha).fit(
            X_train.iloc[tr_idx], y_train.iloc[tr_idx])
        pred = m.predict(X_train.iloc[val_idx])
        ics.append(information_coefficient(pred, y_train.iloc[val_idx])["IC均值"])
    mean_ic = np.mean(ics)
    if mean_ic > best_ic:
        best_ic, best_alpha = mean_ic, alpha

# 用 best_alpha 在全部训练集上重训，最后在测试集上评估一次
```

关键：测试集自始至终只用一次，调参全在训练集内部完成。

## 四、进阶方向地图

### 1. 更好的特征（性价比最高）
- 基本面因子（PE/PB/ROE）：需扩展数据层
- 量价因子（成交量、换手率、资金流）
- 另类数据（新闻情感、研报、龙虎榜）
- 因子的截面标准化、行业中性化

> 再次强调：好特征 >> 好模型。时间优先花在特征上。

### 2. 更好的标签
- 超额收益标签（减去市场，预测相对强弱）
- Triple Barrier 标签（止盈/止损/时间三边界）
- 样本权重（近期样本权重更高）

### 3. 更强的模型
- LightGBM / XGBoost（`pip install` 后加进 `make_model`）
- 模型集成（线性+树+...加权融合）
- 神经网络（见下）

### 4. 深度学习（谨慎）

| 模型 | 适用 | 现实 |
|---|---|---|
| MLP | 表格特征 | 通常打不过 GBDT，不值得 |
| LSTM/GRU | 时序序列 | 理论好，实际易过拟合、难训练 |
| Transformer | 长序列、多资产 | 前沿，需要大量数据和算力 |
| CNN | 把价格图当图像 | 噱头多于实用 |

冷静认知：
> 在表格化的因子数据上，深度学习很少打败 GBDT。
> 深度学习的优势在"原始非结构化数据"：文本、图像、订单流。
> 别因为"深度学习高大上"就上 LSTM，先把线性和 GBDT 做扎实。

### 5. 组合优化
预测只是第一步。从"预测收益"到"实际仓位"还有：
- 均值-方差优化（Markowitz）
- 风险预算、波动率目标
- 换手率约束、交易成本优化
- 风险因子中性化

这是从"信号"到"组合"的工程，本框架目前用最简单的"等权选 top N"。

## 五、一个现实的认知框架

把 ML 量化的难度排序（从易到难、从高性价比到低）：

```
特征工程        ★★★★★ 性价比最高，先做这个
正确的验证      ★★★★★ 不做就是自欺欺人
简单模型(线性)  ★★★★☆ 基准，常常够用
树模型(GBDT)    ★★★☆☆ 锦上添花
超参调优        ★★☆☆☆ 边际收益递减
深度学习        ★☆☆☆☆ 表格数据上很少值得
```

## 六、把这套用到真实数据

模拟数据走通后，换真实数据只需改一行：

```python
# 把
prices = generate_synthetic_prices(symbols, ...)
# 换成
from src.data import load_prices
prices = load_prices(["AAPL","MSFT",...], "2015-01-01", "2024-12-31", market="us")
```

其余完全不变，这就是分层架构的价值。
真实数据的 IC 会比模拟数据低很多（0.02-0.05 是好水平），别被打击，这才是真实战场。
