# 金融数据的坑：数据泄露与过拟合

> 这是整个 ML 量化模块**最重要**的一篇。
> 90% 的"惊人回测"都死在这两个坑里。学会识别它们，比学会任何模型都值钱。

## 坑一：数据泄露（Data Leakage）

定义：**训练时用到了实盘中不可能提前知道的信息。**

后果：回测夏普 3.0，实盘亏到怀疑人生。

### 泄露形式 1：用了未来数据（未来函数）

```python
# ❌ 特征里混入未来
feature = prices.shift(-1)   # 用了明天的价格当今天的特征
```

防范：所有特征只用 `shift(正数)` 或 `rolling`（向过去看）。

### 泄露形式 2：随机划分训练/测试集

这是 ML 新手做量化**最常犯**的错误。

```python
# ❌ 普通 ML 的做法，在金融上是灾难
from sklearn.model_selection import train_test_split
X_tr, X_te = train_test_split(X, y, shuffle=True)  # 随机打乱！
```

为什么错？随机打乱后：
- 2024 年的样本可能进了训练集
- 2020 年的样本进了测试集
- 模型"见过未来"再去预测"过去"
- 测试分数虚高，毫无意义

正确做法：**严格按时间切。**
```python
# ✅ 本框架的做法
from src.ml import time_series_split
X_tr, y_tr, X_te, y_te = time_series_split(X, y, test_size=0.3)
# 训练集全部在测试集之前
```

### 泄露形式 3：标签重叠（最隐蔽）

这个坑连有经验的人都会踩。

假设标签是"未来 5 天收益"：
```
训练集最后一天 t 的标签 = t 到 t+5 的收益
测试集第一天 t+1 的特征 = 用 t+1 及之前的数据
```

→ 训练样本的**标签期** (t~t+5) 和测试样本的**特征期**重叠了！
模型间接"见过"测试集开头的信息。

防范：**embargo（隔离带）**。训练集和测试集之间空出 ≥ horizon 天。

```python
# 本框架默认留隔离带
time_series_split(X, y, test_size=0.3, embargo_days=5)  # 5 = horizon
#                                       ^^^^^^^^^^^^^^^
#                          训练集末尾和测试集开头隔开 5 天
```

代码实现见 `src/ml/dataset.py`：
```python
test_start_idx = min(split_idx + embargo_days, n_dates - 1)
```

### 泄露形式 4：用全样本做预处理

```python
# ❌ 用了整个数据集（含测试集）的统计量
scaler = StandardScaler().fit(X)        # 偷看了测试集的均值/方差
X_tr_scaled = scaler.transform(X_tr)

# ✅ 只用训练集 fit
scaler.fit(X_tr)                        # 只看训练集
X_te_scaled = scaler.transform(X_te)
```

本框架把 StandardScaler 放进 Pipeline，并且只在训练集上 fit，避免这个坑：
```python
Pipeline([("scaler", StandardScaler()), ("model", estimator)])
# fit 时只见训练集，predict 时用训练集学到的均值方差
```

### 泄露形式 5：幸存者偏差
只用"现在还存在的股票"训练——退市/破产的票被排除了。
模型学到的是"幸存者的规律"，实盘要面对所有股票。
防范：用 point-in-time 的历史成分股数据。（本框架数据源有限，未处理）

## 坑二：过拟合（Overfitting）

定义：模型**记住了训练数据的噪声**，而不是学到了普适规律。

症状：训练集表现极好，测试集（样本外）表现差。

### 为什么金融数据特别容易过拟合

1. **样本少**：10 年日线才 2500 天，深度学习动辄要百万样本
2. **噪声大**：信噪比极低，模型容易把噪声当信号
3. **非平稳**：规律本身在变，"学得太死"反而有害
4. **多重检验**：你试了 100 个策略，总有一个"碰巧"很好

### 识别过拟合

```
训练集 IC = 0.30，测试集 IC = 0.02   → 严重过拟合
训练集 IC = 0.06，测试集 IC = 0.05   → 健康
```

**训练和测试差距越大，过拟合越严重。**

### 防过拟合的武器

| 方法 | 原理 | 本框架 |
|---|---|---|
| 正则化 | 惩罚复杂模型（Ridge/Lasso） | `make_model("ridge")` |
| 限制模型复杂度 | 树的 max_depth 调小 | rf/gbdt 默认 depth 3-5 |
| 限制叶子样本数 | min_samples_leaf 调大 | 默认 50 |
| 更多数据 | 样本越多越难过拟合 | 多股票×多年 |
| 减少特征 | 维度灾难 | 只用 10 个精选因子 |
| 交叉验证 | 多折验证更可靠 | `purged_kfold` |
| 简单模型优先 | 奥卡姆剃刀 | 先试线性 |

### 一个反直觉的事实

> 在金融里，**简单模型常常打败复杂模型。**

回顾 `examples/11` 的真实结果（模拟数据）：

| 模型 | 测试集 IC |
|---|---|
| ridge（线性，最简单） | **0.078** |
| rf（随机森林） | 0.057 |
| gbdt（梯度提升） | 0.026 |

最简单的线性模型 IC 最高！因为：
- 信号本身是线性的
- 复杂模型把噪声也学了进去（过拟合）

**默认从最简单的模型开始，只有当它明显不够时才上复杂模型。**

## 坑三：多重检验偏差（别骗自己）

你试了 50 个特征组合、20 个模型、10 组超参数 = 10000 次尝试。
其中"最好"的那个，很可能只是**运气好**，不是真有效。

这叫 **p-hacking / 多重检验**。

防范：
- 减少尝试次数，先有假设再验证
- 用**完全独立**的样本外数据做最终确认（且只用一次）
- 警惕"调到夏普很高"——调出来的高夏普通常是假的
- 用 walk-forward：每段都是真·样本外

## 坑四：前视性最优化（Look-ahead in research）

你用 2015-2024 全部数据"研究"出最好的因子，然后回测 2015-2024。
你已经知道了答案再去考试，当然满分。
正确：在 2015-2020 研究，2021-2024 验证（且不许回头改）。

## 终极检查清单（每次建模必过）

- [ ] 训练集严格在测试集之前？（没随机划分）
- [ ] 训练/测试之间有 embargo 隔离带？（≥ horizon）
- [ ] 预处理只在训练集 fit？（scaler 没偷看测试集）
- [ ] 特征都是当下可知的？（没未来函数）
- [ ] 训练 IC 和测试 IC 差距合理？（没严重过拟合）
- [ ] 跟简单基准比了吗？（买入持有、单因子）
- [ ] 尝试次数有没有失控？（没 p-hacking）
- [ ] 最终验证集只用了一次？

> 记住：在量化 ML 里，**正确的验证方法 > 模型本身**。
> 一个会做正确验证的人，比一个会调 100 个模型但会泄露数据的人强一万倍。
