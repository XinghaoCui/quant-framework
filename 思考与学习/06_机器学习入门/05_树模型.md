# 树模型：决策树、随机森林、GBDT

树模型是量化里**最常用**的非线性模型（比深度学习还常用）。本篇讲清原理和取舍。

## 一、决策树：一切的基础

决策树就是一连串 if-else 问题：

```
            动量 > 0.05?
           /          \
         是            否
        /                \
   波动率 < 0.2?        预测: -1%
    /        \
   是         否
预测:+2%    预测:+0.5%
```

模型自动从数据里学出"问哪些问题、用什么阈值切分"，目标是让每个叶子里的样本尽量"纯"（收益接近）。

### 优点
- 天然处理**非线性**和**特征交互**（"高动量且低波动"是一条路径）
- 不需要标准化（只看相对大小）
- 可解释（能画出树）

### 致命缺点
- **极易过拟合**：一棵深树能把训练集背得一字不差
- **不稳定**：数据微小变化导致树结构剧变

所以**单棵决策树几乎不单独用**，而是组合成森林。

## 二、随机森林（Random Forest）：三个臭皮匠

思想：**训练很多棵不同的树，投票/平均。**

怎么让每棵树"不同"：
1. **行采样**（Bagging）：每棵树用随机抽样的样本子集训练
2. **列采样**：每次分裂只考虑随机的特征子集

最终预测 = 所有树预测的平均（回归）或投票（分类）。

### 为什么有效

单棵树方差大（过拟合），但**很多棵独立的树平均后方差大降**。
就像问 100 个人估体重取平均，比问 1 个人准。

数学上：N 个相关性 ρ 的预测器平均后，方差降到约 $\rho \sigma^2 + \frac{1-\rho}{N}\sigma^2$。
随机化让树之间 ρ 变小，方差降得更多。

### 关键超参数
```python
make_model("rf",
    n_estimators=200,      # 树的数量，越多越稳但越慢
    max_depth=5,           # 每棵树最大深度，越小越不易过拟合
    min_samples_leaf=50,   # 叶子最少样本数，越大越平滑
)
```

> 量化里 max_depth 通常设小（3-6），min_samples_leaf 设大（几十到几百），
> 因为金融数据噪声大，要强力防过拟合。

## 三、GBDT：梯度提升树（通常最强）

随机森林是"并行"种很多独立的树。
GBDT 是"串行"：每棵新树专门修正前面所有树的**残差（错误）**。

```
树1 预测 → 有误差 → 树2 学这个误差 → 还有误差 → 树3 学剩下的 → ...
最终 = 树1 + 树2 + 树3 + ...（加权累加）
```

像考试订正：每次专攻上次的错题，逐步逼近正确答案。

### 为什么常常最强
- 串行修正，拟合能力强
- 工业界 XGBoost / LightGBM / CatBoost 都是 GBDT 的高效实现
- Kaggle 表格数据竞赛**长期霸榜**

### 代价
- 串行训练慢（不能像 RF 那样完全并行）
- 超参数多，调参敏感
- **更容易过拟合**（拟合能力太强是把双刃剑）

### 关键超参数
```python
make_model("gbdt",
    n_estimators=200,      # 树的数量（迭代次数）
    max_depth=3,           # 每棵树很浅（GBDT 用很多浅树）
    learning_rate=0.05,    # 学习率，每棵树贡献多少，小→稳但需更多树
    subsample=0.8,         # 每棵树用 80% 样本，加随机性防过拟合
)
```

**learning_rate 和 n_estimators 是一对**：学习率小，就需要更多树。

### XGBoost / LightGBM 呢？

它们是 GBDT 的高性能实现，更快更强，是实盘和竞赛的主力。
本框架用 sklearn 自带的 `GradientBoostingRegressor`，因为：
- Python 3.14 上 xgboost/lightgbm 的预编译包还不一定就绪
- sklearn 自带的概念完全一样，学会了无缝迁移

扩展时只要 `pip install lightgbm`，在 `make_model` 加一个分支即可。

## 四、树模型 vs 线性模型

| 维度 | 线性模型 | 树模型 |
|---|---|---|
| 关系 | 只能线性 | 非线性、交互 |
| 标准化 | 必须 | 不需要 |
| 可解释 | 强（看系数） | 中（看重要性） |
| 过拟合 | 较难 | **容易** |
| 速度 | 快 | 慢（尤其 GBDT） |
| 异常值 | 敏感 | 稳健 |
| 小样本 | 更稳 | 易过拟合 |

## 五、实战：复杂不等于更好

`examples/11` 在模拟数据上对比三个模型，真实结果：

```
           IC均值      ICIR    多空价差日均
ridge    0.078    0.428    0.0086   ← 最简单的线性，最好！
rf       0.057    0.309    0.0068
gbdt     0.026    0.145    0.0029   ← 最复杂的，最差
```

**线性模型 IC 最高，GBDT 最差！** 为什么？
- 模拟数据的信号本来就是线性的（GBM drift）
- 树模型把噪声也学了进去 → 过拟合 → 样本外更差

这完美印证了 [03_金融数据的坑](./03_金融数据的坑.md) 的教训：
> 复杂模型不一定更好，尤其当真实信号简单、数据噪声大时。

**真实市场里**，树模型通常因能捕捉非线性而略胜线性，
但前提是你有足够多的好特征和样本，且做好了防过拟合。

## 六、特征重要性

树模型能输出每个特征的重要性（它在多少次分裂里被用到、降低了多少误差）。

`examples/11` 随机森林的真实输出：
```
mom_252_21     0.326   ← 12-1 动量贡献最大
vol_63         0.194
mom_63         0.150
vol_21         0.075
...
```

用途：特征筛选、理解模型、查 bug（见 [02_特征工程与标签](./02_特征工程与标签.md)）。

> 注意：树的特征重要性会**高估高基数特征**，且相关特征会"分摊"重要性。
> 更可靠的是 permutation importance（打乱某特征看性能下降多少）。

## 七、如何选模型（决策流程）

```
1. 先用 ridge 线性模型打基准
        ↓
2. 基准 IC 还行？ → 可能就够了，线性最稳
        ↓ 不够
3. 试 random forest（更稳健的非线性）
        ↓ 还想要更强
4. 试 gbdt（最强但最易过拟合，小心调参）
        ↓
5. 每一步都看【测试集 IC】，且和上一步比，不是越复杂越选
```

## 八、小结

- 决策树 = if-else 规则，单用易过拟合
- 随机森林 = 很多独立树平均，稳健，并行快
- GBDT = 串行修正残差，最强但易过拟合
- 树模型擅长非线性，但金融里**简单常常更好**
- 永远看测试集 IC 做对比，警惕复杂度
- 实盘主力是 LightGBM/XGBoost，概念和本框架一致
