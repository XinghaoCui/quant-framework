# 模型评估：用量化的方式，不是普通 ML 的方式

> 普通 ML 看准确率、R²。在量化里，这些几乎没用。
> 量化关心的是：模型的预测排序，能不能转化成钱。

## 一、为什么不能用普通指标

### R² 在量化里低得吓人

普通回归看 R²（解释了多少方差），通常期望 0.7+。
但量化里预测收益的 R² 经常是 0.001~0.01，甚至为负。

新手看到会以为"模型没用"。其实：
> 信噪比极低的金融数据，R²=0.01 已经可能是赚钱的策略。
> 因为放大到几千只票、几千个交易日，微弱的优势会累积成可观收益。

### 准确率也会误导

分类准确率 51% 听起来跟瞎猜差不多。
但如果你每次都下注，51% 胜率 + 合理盈亏比 = 长期稳定盈利（赌场逻辑）。
而且准确率没告诉你"预测最强的那些股票对不对"，而那才是你重仓的。

## 二、量化的核心评估：IC（信息系数）

### 定义

IC = 每个截面（每天），预测值和真实未来收益的相关系数，再对时间取平均。

$$
IC_t = \text{corr}\big(\text{prediction}_t, \text{actual return}_t\big)
$$

$$
\overline{IC} = \text{mean}_t(IC_t)
$$

它衡量的是：模型的排序和真实收益的排序有多一致。

### 两种 IC
- Pearson IC：普通相关系数，对极端值敏感
- Rank IC (Spearman)：用排序算相关，更稳健，业界更常用

本框架默认 Rank IC：
```python
information_coefficient(pred, actual, method="spearman")
```

### IC 判读

| IC 均值 | 判断 |
|---|---|
| < 0.02 | 基本没用 |
| 0.02 ~ 0.05 | 可用（真实市场常见水平） |
| 0.05 ~ 0.10 | 不错 |
| > 0.10 | 很强（真实市场罕见，模拟数据才容易达到） |

### ICIR：IC 的稳定性

光 IC 高不够，还要稳定。
$$
ICIR = \frac{\text{mean}(IC_t)}{\text{std}(IC_t)}
$$

像 IC 的"夏普比率"。ICIR > 0.5 说明因子/模型稳定有效。

`examples/10` 真实输出：
```
IC均值: 0.1037     ← 排序能力强
IC标准差: 0.1776
ICIR: 0.5839       ← 稳定性不错
IC>0占比: 0.7065   ← 70% 的天 IC 为正
```

## 三、分组回测（Quantile Backtest）

IC 是个统计量，分组回测更直观：按预测值把股票分组，看各组真实收益。

做法：
1. 每天把股票按模型预测值分成 5 组（Q1 最低预测 ... Q5 最高预测）
2. 算每组的平均真实收益
3. 看是否单调：Q5 > Q4 > Q3 > Q2 > Q1

`examples/10` 真实输出：
```
        平均收益       累计收益
Q1 -0.005040      -0.97   ← 预测最差的，确实跌最多
Q2 -0.001745      -0.73
Q3  0.000558       0.33
Q4  0.004356      17.34
Q5  0.007804     198.17   ← 预测最好的，确实涨最多

多空价差（日均）: 0.01284
```

### 怎么看
- 单调性：组间收益单调递增 = 模型排序有效（最理想）
- 多空价差：Q5 - Q1 的收益，越大越好，这是"多空策略"的收益来源
- Q5 是否最高：实盘往往只买 Q5（最看好的），它必须真的好

单调性比"绝对收益"更重要，它说明模型的整个排序都对，不是碰运气。

## 四、IC vs 分组回测 vs 真实回测

三个层次，逐步接近实盘：

| 层次 | 衡量 | 优点 | 局限 |
|---|---|---|---|
| IC | 排序相关性 | 快、统计严谨 | 不含交易成本、仓位 |
| 分组回测 | 分层收益 | 直观看单调性 | 不含成本、调仓细节 |
| 真实回测 | 净值曲线 | 含成本、换手、夏普 | 最慢 |

正确流程：IC 筛因子/模型 → 分组验证单调 → 真实回测看能否赚钱（扣成本）。

本框架三个都有：
- IC + 分组：`src/ml/evaluation.py`
- 真实回测：`MLStrategy` 接入 `Backtester`（`examples/12`）

## 五、最重要的评估：和基准比

再高的 IC，打不过简单基准就没价值。

`examples/12` 真实结果：

| 策略 | 夏普 | 最大回撤 |
|---|---|---|
| GBDT ML 策略 | 0.56 | -17.2% |
| 等权买入持有（基准） | 1.36 | -7.4% |

ML 策略全面输给等权买入持有。

为什么有 IC 还输？
- IC 衡量"排序对不对"，但选股集中（只买 30%）放大了波动
- 等权买入持有充分分散，模拟数据里所有票都涨，基准天然强
- 交易成本、调仓时机进一步拖累

教训（本模块反复强调）：
> IC 高 ≠ 策略好。必须做真实回测、扣成本、和基准比。
> 一个打不过买入持有的"高 IC 模型"，是自我欺骗。

## 六、评估流程清单

```
1. 训练集 IC vs 测试集 IC
   → 差距大？过拟合，回去防过拟合

2. 测试集 IC > 0.02？ICIR > 0.5？
   → 否则模型基本没用

3. 分组回测单调吗？Q5 真的最好吗？
   → 不单调说明排序不可靠

4. 真实回测（扣成本）夏普如何？
   → 这才是钱

5. 打得过买入持有/单因子基准吗？
   → 打不过就别用
```

## 七、常见评估错误

- 只看训练集表现（必须看样本外）
- 用准确率/R² 下结论（量化要看 IC）
- 只看 IC 不做回测（IC 高也可能亏钱）
- 不和基准比（孤立的高夏普没意义）
- 反复调参直到测试集好看（测试集被你"用脏"了，成了第二个训练集）

## 八、小结

- 量化不看准确率/R²，看 IC（排序相关性）
- IC 均值看强度，ICIR 看稳定性
- 分组回测看单调性，多空价差是收益来源
- 三层评估：IC → 分组 → 真实回测（扣成本）
- 终极标准：打不打得过简单基准
- IC 高 ≠ 赚钱，务必真实回测 + 对比基准
