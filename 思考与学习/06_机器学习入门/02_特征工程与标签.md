# 特征工程与标签构造

> "Garbage in, garbage out." 特征决定了模型的上限，模型只是逼近这个上限。
> 在量化里，**特征工程比模型选择重要 10 倍。**

## 一、什么是特征工程

把原始数据（价格、成交量、财报）变成模型能用的**特征矩阵 X**。

在我们的框架里，特征就是**因子**。`src/ml/features.py` 用了 10 个特征：

```python
feats["mom_21"]      = 1 月动量
feats["mom_63"]      = 3 月动量
feats["mom_252_21"]  = 经典 12-1 月动量
feats["vol_21"]      = 1 月波动率
feats["vol_63"]      = 3 月波动率
feats["reversal_5"]  = 1 周反转
feats["rsi_14"]      = RSI
feats["px_vs_sma20"] = 价格相对 20 日均线位置
feats["px_vs_sma60"] = 价格相对 60 日均线位置
feats["sma20_slope"] = 20 日均线斜率（趋势方向）
```

这些都来自之前写的因子库 `src/factors/`，ML 模块直接复用——
**这就是分层架构的好处：因子既能单独做策略，又能当 ML 特征。**

## 二、特征工程的核心原则

### 原则 1：特征必须是"当下可知"的

> 在 t 时刻算特征，只能用 t 时刻及之前的数据。

```python
# ✅ 正确：动量用过去的价格
mom = prices / prices.shift(21) - 1

# ❌ 错误：用了未来价格
bad = prices.shift(-5) / prices - 1   # 这是标签，不是特征！
```

混淆这一点 = 数据泄露 = 回测完美、实盘归零。

### 原则 2：横截面可比

不同股票的特征要在同一尺度上可比。
- 用"收益率"（百分比），不用"价格"（绝对值）—— 茅台 1700 元 vs 某股 5 元，价格没法比，但收益率可以
- 必要时做横截面标准化（z-score / rank）

### 原则 3：处理缺失值
- 因子前期天然有 NaN（如 60 日均线前 59 天没值）→ 这些样本直接丢弃
- 中途偶发缺失 → 可以前向填充或丢弃
- **不要用 0 填充**——0 是有意义的因子值，会污染

### 原则 4：避免特征爆炸
- 特征不是越多越好，相关性高的特征是冗余
- 金融数据样本少，特征太多 → 维度灾难 → 过拟合
- 宁可 10 个好特征，不要 100 个噪声特征

## 三、标签构造（y）

标签 = 我们想预测的目标 = **未来收益**。

```python
# src/ml/features.py:build_label
forward_return = prices.shift(-horizon) / prices - 1.0
```

关键是 `shift(-horizon)`：负号表示"未来"。
```
forward_return(t) = P(t+5) / P(t) - 1   # t 之后 5 天的收益
```

### 标签的几个选择

#### 1. 预测窗口 horizon
- horizon=1：预测明天，噪声极大
- horizon=5：预测一周，常用，平衡信噪比和时效
- horizon=21：预测一月，信号更稳但调仓慢

#### 2. 回归 vs 分类标签
```python
# 回归：连续收益率
y = forward_return

# 分类：涨跌方向
y = (forward_return > 0).astype(int)
```

#### 3. 进阶标签（本框架未实现，扩展方向）
- **超额收益**：减去市场平均，预测"相对强弱"而非绝对涨跌（更稳）
- **风险调整收益**：收益 / 波动率
- **三分类**：大涨/震荡/大跌（过滤掉小波动噪声）
- **Triple Barrier**（López de Prado）：用止盈/止损/时间三个边界打标签

## 四、X 和 y 的时间对齐（最易错）

这张图是理解金融 ML 的关键：

```
时间轴 ──────────────────────────────────────▶
         t-60 ... t-1   [t]   t+1 ... t+5
         └──────────┘    │    └──────────┘
          算特征用这段    │     标签是这段
          （过去）       决策点   （未来）
                         │
              在 t 点：用过去算 X，预测未来 y
```

- **特征 X(t)**：用 t 及之前的数据
- **标签 y(t)**：t 到 t+horizon 的未来收益
- 训练时：模型学 X(t) → y(t) 的关系
- 预测时：给新的 X(t')，预测它的 y

`make_dataset` 把这些对齐好，并丢掉两头的 NaN：
```python
X, y = make_dataset(prices, horizon=5, kind="return")
```

## 五、长表 vs 宽表

回测引擎用**宽表**（index=date, columns=symbol）。
但 sklearn 要**长表**（每行一个样本）。

```
宽表（价格）：              长表（ML 样本）：
        AAPL  MSFT          (date, symbol)  mom_21  vol_21  ...
2020-01  100   200          (2020-01,AAPL)   0.05    0.12
2020-02  110   190          (2020-01,MSFT)   0.03    0.15
                            (2020-02,AAPL)   0.10    0.11
                            ...
```

转换用 pandas 的 `stack()`：
```python
s = wide_factor.stack()   # 宽表 → 长表，MultiIndex=(date, symbol)
```

预测完再 `unstack()` 转回宽表喂给回测引擎。
`src/ml/features.py` 和 `ml_strategy.py` 处理了这些来回转换。

## 六、特征重要性：模型告诉你哪些特征有用

训练后，模型能告诉你每个特征的贡献。
看 `examples/12` 的真实输出（GBDT）：

```
mom_252_21     0.195   ← 12-1 动量最重要
vol_63         0.148
mom_63         0.144
vol_21         0.115
...
reversal_5     0.064
```

用途：
- **特征筛选**：删掉重要性极低的特征，减少过拟合
- **理解模型**：知道它在"靠什么赚钱"
- **怀疑检查**：如果某个不该重要的特征排第一，可能有 bug 或泄露

线性模型看系数绝对值，树模型看 `feature_importances_`，
统一接口在 `ModelWrapper.feature_importance()`。

## 七、实战检查清单

写完特征后，问自己：
- [ ] 每个特征都是 t 时刻可知的吗？（没用未来数据）
- [ ] 标签是 t 之后的收益吗？（shift 是负的）
- [ ] X 和 y 之间有没有时间重叠？（标签期不能进特征）
- [ ] 缺失值处理对吗？（丢弃，不是填 0）
- [ ] 特征横截面可比吗？（用收益率不用价格）
- [ ] 特征数量合理吗？（不要爆炸）
