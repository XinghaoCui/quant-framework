# 滚动重训实战（Walk-Forward）

第 7 篇提到 walk-forward 是最值得做的扩展。本篇把它实现并讲透。
配套代码 `src/ml/walkforward.py`，示例 `examples/13_ML_滚动重训.py`。

## 一、为什么需要滚动重训

"训练一次"的模型有个致命假设：**市场规律永远不变。**

但市场是**非平稳**的：
- 因子会衰减（被套利者拉平）
- régime 会切换（牛市/熊市/震荡逻辑完全不同）
- 黑天鹅改变相关性结构（2020 疫情、2022 加息）

> 用 2015 年学的模型预测 2024 年 = 刻舟求剑。

实盘里没人训练一次就不管了。标准做法是**每隔一段时间用最新数据重训**。
walk-forward 就是在回测里**忠实模拟这个过程**。

## 二、核心思想

```
时间轴 ─────────────────────────────────────────▶
|--- 初始训练 ---|e|-- 预测段1 --|
|------ 训练 -----|e|-- 预测段2 --|
|-------- 训练 ----|e|-- 预测段3 --|
                                  ...
（e = embargo 隔离带）
```

每一轮：
1. 用"截至当前"的数据训练模型
2. 跳过 embargo 天（防标签泄露）
3. 预测接下来 retrain_freq 天
4. 时间前移 retrain_freq，重训，循环

所有预测段无缝拼接 → 一条完整的、**每一段都是真·样本外**的预测序列。

## 三、两种窗口模式

### expanding（扩展窗口）
训练集**起点固定**，终点不断后移 → 训练集越来越大。

```
轮1: [2016 ────── 2017]
轮2: [2016 ──────────── 2018]
轮3: [2016 ──────────────────── 2019]
```

- ✅ 数据利用最充分，样本越来越多
- ❌ 很久以前的旧规律一直留在训练集里，钝化对新变化的反应

### rolling（滚动窗口）
训练集**长度固定**，起点终点一起后移 → 永远只用最近一段。

```
轮1: [2016 ── 2017]
轮2:    [2017 ── 2018]
轮3:       [2018 ── 2019]
```

- ✅ 能"遗忘"过时规律，更快适应市场变化
- ❌ 丢弃了老数据，样本量受限，可能更不稳

### 怎么选
- 市场结构相对稳定、想要充分数据 → expanding
- 市场快速变化、因子衰减快 → rolling
- 不确定时两个都跑，看样本外表现

## 四、防泄露：embargo 隔离带

和 [03_金融数据的坑](./03_金融数据的坑.md) 讲的一样：
标签是 horizon 天后的收益，所以训练集最后几天的标签会"看到"预测段开头。
必须在训练止和预测起之间留 ≥ horizon 天的隔离带。

代码里默认 `embargo_days = horizon`：
```python
pred_start_idx = i + embargo   # 预测从训练止 + embargo 之后开始
```

## 五、代码用法

### 方式一：直接用函数
```python
from src.ml import walk_forward_predict

predictions, log = walk_forward_predict(
    prices,
    model_name="ridge",
    horizon=5,
    initial_train_days=504,   # 初始训练 2 年
    retrain_freq=126,         # 每半年重训
    window="expanding",       # 或 "rolling"
    verbose=True,             # 打印每轮进度
)
print(log)   # 重训日志：每轮的训练区间、预测区间、样本数
```

### 方式二：通过 MLStrategy（推荐，能接回测）
```python
from src.ml.ml_strategy import MLStrategy

strat = MLStrategy(
    model_name="ridge",
    retrain_freq=126,         # 关键：设了就是滚动重训；None 就是训练一次
    window="expanding",
    initial_train_days=504,
)
weights = strat.generate_weights(prices)
# 再丢给 Backtester 即可，和其他策略一样
print(strat.walk_log_)        # 重训日志
```

**一个参数 `retrain_freq` 切换两种模式**，这就是好接口的价值。

## 六、实战结果解读（重要）

`examples/13` 在模拟数据上对比三种模式，真实输出：

| 模式 | 预测起止 | 预测样本 | IC均值 | 夏普 |
|---|---|---|---|---|
| 训练一次 | 2020-07 ~ 2024-12 | 34830 | 0.0889 | **1.735** |
| expanding | 2018-01 ~ 2024-12 | 54660 | **0.0921** | 0.906 |
| rolling | 2018-01 ~ 2024-12 | 54660 | 0.0875 | 0.901 |

### 关键观察 1：walk-forward 样本外区间长得多
- 训练一次：只能从 2020-07 开始预测（前 50% 拿去训练了）
- walk-forward：从 2018 就开始预测，覆盖 **7 年** 而非 4 年

→ 更长、更多样的样本外检验，结论更可信。

### 关键观察 2：walk-forward 夏普"看着更低"，但这恰恰更诚实

新手会以为"训练一次夏普 1.7 > walk-forward 0.9，所以训练一次更好"。**错！**

真相：
- "训练一次"只覆盖了 2020 后那段相对顺的行情，夏普虚高
- walk-forward 覆盖了 2018 震荡、2020 暴跌等**更难的时期**，夏普更低但**更接近真实实盘表现**

> 一个在更长、更难样本外区间上夏普 0.9 的结果，
> 比一个只在"精选的好时段"上夏普 1.7 的结果**可信得多**。

这正是 walk-forward 的价值：**它不让你只在容易的时段上自我感觉良好。**

### 关键观察 3：重训日志让过程透明
```
重训序号  训练起      训练止      训练样本  预测起      预测止
   1     2016-01-19  2017-12-25  15150    2018-01-01  2018-06-25
   2     2016-01-19  2018-06-19  18930    2018-06-26  2018-12-18
   ...
  15     2016-01-19  2024-09-27  68070    2024-10-04  2024-12-24
```
expanding 模式下训练样本从 15150 一路涨到 68070（窗口在扩大），
每段预测都紧接上一段，无缝覆盖。这种透明性便于排查和信任。

## 七、参数选择经验

| 参数 | 含义 | 经验值 |
|---|---|---|
| initial_train_days | 初始训练窗口 | 504（2年）~ 756（3年），太短模型学不动 |
| retrain_freq | 重训频率 | 21（月）/ 63（季）/ 126（半年）/ 252（年） |
| window | 窗口模式 | 先 expanding 打底，再试 rolling |
| rolling_window_days | rolling 训练长度 | 504~756，太短样本不够 |
| embargo_days | 隔离带 | = horizon（默认） |

重训越频繁，越能适应变化，但：
- 计算成本越高（每轮训一个模型）
- 每次训练样本可能变少（rolling）
- 过于频繁会让模型"追噪声"

## 八、性能注意

walk-forward 要训练很多次模型：
- ridge/lasso：极快，几十轮也就几秒
- rf：中等
- **gbdt：慢**，每轮一个 GBDT，几十轮可能几分钟

建议：
- 研究阶段用 ridge 快速迭代
- 确定流程后再用 gbdt 跑最终结果
- 真实大规模可考虑 LightGBM（带早停，更快）

## 九、进阶扩展方向

### 1. 滚动调参
每轮重训时，在该轮训练集内部用 `purged_kfold` 选超参数，
而不是全程用固定超参。更贴近实盘，但慢很多。

### 2. 样本权重
近期样本权重更高（市场更可能延续近期规律）。
sklearn 的 `fit(sample_weight=...)` 支持。

### 3. 模型监控
记录每轮的训练 IC vs 预测 IC，画成时间序列。
如果预测 IC 持续衰减 → 因子失效信号，该换因子了。

### 4. 滚动特征重要性
记录每轮的 feature_importance，看哪些因子的重要性随时间变化。
这能发现"因子轮动"现象。

## 十、小结

- 市场非平稳 → 必须滚动重训，"训练一次"只适合教学
- expanding 数据充分，rolling 适应快，按市场特性选
- embargo 隔离带防标签泄露（和普通时序划分一致）
- `MLStrategy(retrain_freq=126)` 一个参数切换模式
- **walk-forward 夏普看着低是因为样本外更长更诚实**，不是模型更差
- 这是从"玩具回测"走向"可信回测"的关键一步

做完这个，你已经掌握了金融 ML 工程里**最重要的实践**之一。
