# 把组合优化接入策略

> 前 5 篇讲了概念和方法, 这一篇把它们**端到端**接入回测策略。
> 跑一遍 `examples/32`, 你就掌握了完整的"打分 → 选股 → 优化 → 回测" 流水线。

## 一、完整流水线长什么样

任何一个量化策略, 拆开都是这四步:

```
┌──────────────────────────────────────────────────────────────┐
│  1. 信号生成 (Signal Generation)                              │
│     原始价格 → 因子打分 / ML 预测 / 自定义信号                  │
│     ↓                                                         │
│  2. 选股 (Stock Selection)                                    │
│     每天按信号选 top N% 进入候选池                              │
│     ↓                                                         │
│  3. 组合优化 (Portfolio Optimization)                         │
│     在 top N 内部按某种规则分配权重 (等权 / min_var / risk_parity) │
│     ↓                                                         │
│  4. 调仓与回测 (Rebalancing & Backtest)                       │
│     按调仓周期生成权重矩阵, 交给回测引擎                          │
└──────────────────────────────────────────────────────────────┘
```

前 7 章主要在 1+2 步发力 (因子、ML), 第 4 步引擎一直能跑。
**本章把第 3 步从"等权" 升级到"组合优化"**, 完成最后一块拼图。

## 二、OptimizedStrategy 设计思路

为了把组合优化接入策略层, 框架提供了 `OptimizedStrategy`:

```python
class OptimizedStrategy(Strategy):
    def __init__(
        self,
        scores: pd.DataFrame,      # 每天每只票的打分
        optimizer: str = "min_variance",   # 用哪种优化
        top_pct: float = 0.3,      # 选打分前 30%
        rebalance_days: int = 21,  # 月度调仓
        lookback_days: int = 60,   # 用 60 天历史估 Σ
        max_weight: float | None = None,
    ):
        ...
```

**关键设计选择**:
1. **scores 跟 prices 分开传**: scores 可以来自任何源 (因子合成、ML、自定义), 不绑死
2. **优化只支持"只看 Σ" 的方法**: equal / inverse_vol / risk_parity / min_variance
   - 不支持 mean_variance / max_sharpe 是因为它们需要 μ, 而 μ 的估计是单独的复杂工程, 不该混在策略里
3. **数值不稳时退化等权**: 历史不足或求解失败时, fallback 到等权, 保证回测不中断
4. **opt_log_ 记录调仓诊断**: 每次调仓的 vol、收敛状态都存着, 方便排查

## 三、5 步代码完整示例

```python
import pandas as pd
from src.data import generate_synthetic_prices
from src.engine import Backtester
from src.factors import momentum, reversal, volatility
from src.strategies.multi_factor import cross_section_zscore
from src.strategies import OptimizedStrategy

# 1. 拿数据
prices = generate_synthetic_prices(
    [f"STK{i:02d}" for i in range(30)],
    "2018-01-01", "2024-12-31", seed=42,
)

# 2. 算多个因子, 截面 zscore 后等权合成综合分
factors = {
    "momentum":  momentum(prices, lookback=252, skip=21),
    "reversal":  reversal(prices, window=21),
    "lowvol":   -volatility(prices, window=60),
}
zs = [cross_section_zscore(f.reindex_like(prices)) for f in factors.values()]
composite = sum(zs) / len(zs)

# 3. 构造策略: 选 top 30%, 在内部跑最小方差
strat = OptimizedStrategy(
    scores=composite,
    optimizer="min_variance",
    top_pct=0.3,
    rebalance_days=21,
    lookback_days=60,
    max_weight=0.4,   # 防止过度集中
)

# 4. 生成权重并回测
weights = strat.generate_weights(prices)
bt = Backtester(commission=0.0003, slippage=0.0001)
result = bt.run(prices, weights)
print(result.summary())

# 5. 查看调仓日志
print(strat.log_dataframe().tail())
```

## 四、真实跑出的对比 (来自 examples/32)

模拟数据上 30 只票 7 年, 5 种策略横比:

```
                     年化收益    夏普比率    最大回撤  Calmar比率  平均换手率(单边)
single_momentum    22.75%     2.30        -8.92%    2.55          0.62%
multi_equal         6.44%     0.73       -20.20%    0.32          2.13%
multi_inv_vol       8.13%     0.99        -9.77%    0.83          1.77%
multi_risk_parity   8.43%     1.02       -10.15%    0.83          1.83%
multi_min_var       7.14%     0.83       -12.25%    0.58          2.01%
```

关键观察:

### 1. 同样的打分, 优化后回撤大幅改善
- `multi_equal` 回撤 -20%
- 加优化后回撤都缩到 -10% 左右
- **这就是组合优化的价值**: 不增 alpha 但降风险

### 2. inverse_vol 和 risk_parity 几乎平手
模拟数据股票相关性低, 两者差异不明显。 真实 A 股相关性较高时差异会更大。

### 3. min_var 在这次没赢
有时候 min_var 过于集中, 反而吃亏。 加 max_weight 约束后稳一点。

### 4. single_momentum 一枝独秀
**模拟数据上**单因子动量比所有多因子组合还猛。 这恰恰说明:
**模拟数据的统计特性不能完全反映真实市场**, 真实 A 股上单因子很少能这么强势。

## 五、调仓日志怎么用

`OptimizedStrategy.log_dataframe()` 返回每次调仓的诊断信息:

```python
print(strat.log_dataframe().tail())
#                method       n_top  expected_vol  success
# date
# 2024-09-10    min_variance     9      0.0089       True
# 2024-10-09    min_variance     9      0.0091       True
# 2024-11-07    min_variance     9      0.0085       True
# 2024-12-06    min_variance     9      0.0087       True
```

用途:
- 看 `expected_vol` 的时序: 是不是某段时间波动突变?
- 看 `success` 是不是都 True: False 表示优化器没收敛, 应该排查
- 看 `n_top` 是否稳定: 不稳定可能 scores 在某些日期缺失严重

## 六、扩展方向

本框架的 `OptimizedStrategy` 是入门级实现。 如果你想做实战级:

### 1. 加 shrinkage 协方差
不直接用 `returns.cov()`, 而是用 Ledoit-Wolf:
```python
from sklearn.covariance import LedoitWolf
cov_shrunk = LedoitWolf().fit(hist).covariance_
```
然后传给优化函数。

### 2. 加行业 / 风格中性化
约束"金融行业总权重 ≤ 30%"。 需要先获取行业归类数据。

### 3. 加换手率约束
减少调仓震荡:
$$
\sum |w_t - w_{t-1}| \leq T
$$
在 SLSQP 里加这个非线性约束即可。

### 4. 用 ML 预测的 μ + Black-Litterman
把 06 章的 ML 预测结果作为"主观观点", 用 BL 模型修正 μ, 然后跑 max_sharpe。

### 5. 多周期优化 (Multi-Period)
现在是单期优化 (每个调仓日独立). 多期优化能考虑"今天的决策对明天的影响", 更接近实盘。
