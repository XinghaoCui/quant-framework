# 最小方差组合

> 不会预测未来收益的人, 最理性的选择就是"找波动最小的组合"。
> 本篇讲: 为什么、怎么求、有什么坑。

## 一、一句话直觉

> 既然我不知道哪只股票未来会涨, 那就找一组让"组合波动最小" 的权重。

从贝叶斯先验的角度看, 这个论证挺优雅:
- 如果你对所有股票的期望收益没有任何信息, 最合理的假设是它们都一样
- 期望收益都一样时, 唯一区别就是波动 → 波动最小就是最优

所以最小方差组合的精神是不预测、不下注, 纯粹避险。

## 二、数学描述

求 w 让 w' Σ w 最小, 满足 sum(w) = 1, w ≥ 0:

$$
\min_{\mathbf{w}} \mathbf{w}^T \Sigma \mathbf{w}
$$

约束:
- $\mathbf{1}^T \mathbf{w} = 1$ (权重和为 1)
- $w_i \geq 0$ (可选: 不允许做空)
- $w_i \leq w_{max}$ (可选: 单票上限)

## 三、解析解 vs 数值解

### 解析解 (无 ≥0 约束)
若不限做空, 有解析公式:
$$
\mathbf{w}^* = \frac{\Sigma^{-1} \mathbf{1}}{\mathbf{1}^T \Sigma^{-1} \mathbf{1}}
$$

公式直觉: $\Sigma^{-1}$ "拆开" 协方差关系, 把权重分给那些"相对其他票最不相关、波动又小" 的资产。

### 数值解 (有 ≥0 约束)
加了 w ≥ 0 后没有解析解, 用二次规划 / SLSQP 数值求解。
本框架用 scipy 的 SLSQP:

```python
from scipy.optimize import minimize

res = minimize(
    fun=lambda w: w @ Sigma @ w,
    x0=np.ones(n) / n,
    method="SLSQP",
    bounds=[(0, 1)] * n,
    constraints=[{"type": "eq", "fun": lambda w: w.sum() - 1.0}],
)
```

完整代码见 `src/portfolio/optimizer.py:min_variance_weights`。

## 四、用法

```python
from src.portfolio import min_variance_weights

# 输入: 历史日收益 DataFrame (date × symbol)
returns = prices.pct_change().dropna()

res = min_variance_weights(returns)
print(res.weights)
# S0    0.10
# S1    0.30
# S2    0.05
# S3    0.40
# S4    0.15
# dtype: float64

print(f"组合日波动: {res.expected_vol:.4f}")
```

或者直接对协方差矩阵:
```python
cov = returns.cov()
res = min_variance_weights(cov)
```

## 五、判读: 最小方差的典型权重特征

跑 `examples/30_组合优化对比.py` 你会看到类似:

```
              equal   inverse_vol   risk_parity   min_variance
STK00       0.1667        0.1989        0.1874        0.2228
STK01       0.1667        0.1609        0.1635        0.1370
STK02       0.1667        0.1398        0.1406        0.1247
STK03       0.1667        0.1943        0.1869        0.2246
STK04       0.1667        0.1653        0.1761        0.1672
STK05       0.1667        0.1409        0.1455        0.1237
```

最小方差的特征:
- 波动最低的票权重最高 (STK00、STK03)
- 波动最高的票权重最低 (STK02、STK05)
- 权重比反波动率还更极端 (因为它还利用了相关性, 不只是波动)

集中度对比:
```
              组合年化波动      HHI (集中度)
equal           0.1203         0.1667
min_variance    0.1166         0.1777   ← 最低波动, 但稍微集中
```

组合波动率比等权低, 这是它的目标。 但集中度更高, 意味着更依赖少数几只票, 实战要警惕。

## 六、坑 1: 极度依赖 Σ 估计的准确性

最小方差对 Σ 非常敏感:
- Σ 里某只票的方差估低了 0.1% → 这只票权重可能从 5% 涨到 30%
- Σ 里某两只票的相关性估高了 0.1 → 它俩权重可能都降一截

所以历史窗口短时, 最小方差权重会剧烈漂移, 实战不可用。

解决方案:
1. lookback 取长一点 (>= 120 天)
2. 用 shrinkage 协方差 (本框架未实现)
3. 加 max_weight 约束防止极端集中

## 七、坑 2: 容易集中到少数股票

如果你的票池里有几只波动极低、互相又低相关的股票, 最小方差会全压它们, 其他都给 0。

例子: 模拟 30 只股票, 但其中 5 只波动极低且互相无关, 跑最小方差:
```
S00: 0.20
S01: 0.18
S02: 0.22
S03: 0.21
S04: 0.19
其他 25 只: 几乎都是 0
```

这违背"分散投资"的初衷。 解决方案 (本框架已实现):
```python
res = min_variance_weights(returns, max_weight=0.10)  # 单只不超过 10%
```

## 八、坑 3: 完全不利用你的"选股信号"

最小方差只看协方差, 不看你给的因子打分。 这意味着:
- 你辛苦研究的动量因子, 没用上
- 你的 ML 模型预测, 没用上

最小方差适合"我选了 N 只股票, 但不知道哪只更好" 的场景。
如果你有信号, 应该用均值方差或下面要讲的"分层做法":
1. 用打分选出 top N
2. 在 top N 内部用最小方差分仓

`OptimizedStrategy` 实现的就是这个分层逻辑。

## 九、什么时候用最小方差

适用场景:
- 没有可信的收益预测
- 追求最低组合波动 (大资金、保守资金)
- 想要一个理论清晰、参数少的方法

不适用场景:
- 你的选股信号很强 (浪费了)
- 票池里波动差异极大 (会过度集中)
- Σ 估计不可靠 (短历史)

## 十、小结

- 最小方差 = "我不预测收益, 只想最稳"
- 求 w 让 w'Σw 最小, 用 scipy SLSQP 求解
- 优点: 理论清晰、参数少、对没有信号的人最优
- 缺点: 对 Σ 敏感、容易集中、忽略选股信号
- 实战: 加 max_weight 防集中, 用足够长的 lookback (>=120 天)
- 接口: `min_variance_weights(returns)` 或 `OptimizedStrategy(scores, optimizer="min_variance")`
