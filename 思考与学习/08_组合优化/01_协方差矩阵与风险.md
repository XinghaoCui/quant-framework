# 协方差矩阵与风险

> 组合优化最核心的输入是**协方差矩阵 Σ**。
> 本篇用最直观的方式讲清: Σ 是什么、为什么必须用它、组合风险怎么算。

## 一、什么是协方差矩阵

假设你有 3 只股票 A、B、C, 协方差矩阵长这样:

```
       A      B      C
   ┌                      ┐
A  │ 0.04   0.02   0.001 │
B  │ 0.02   0.09   0.005 │
C  │ 0.001  0.005  0.16  │
   └                      ┘
```

每个元素 Σᵢⱼ 表示**股票 i 和 j 的收益协方差** (单位: 日收益的协方差, 数值很小很正常)。

读法:
- **对角线** Σᵢᵢ = 第 i 只股票自己的方差 (波动率的平方)
  - Σ_AA = 0.04 → 日波动率 = √0.04 = 0.2 (= 20%/天, 已经很高了, 这里用了夸张数值便于讲解)
- **非对角** Σᵢⱼ = i 和 j 收益的协方差
  - Σ_AB = 0.02 → A 和 B 大体同涨同跌
  - Σ_AC = 0.001 → A 和 C 几乎无关

可以转成更直观的**相关系数矩阵**:

```
           A     B     C
A        1.00  0.33  0.01
B        0.33  1.00  0.04
C        0.01  0.04  1.00
```

相关系数 = 协方差 / (σᵢ × σⱼ), 范围 [-1, 1]。

## 二、为什么"组合风险" 必须用协方差矩阵

直觉上, 一个组合的风险应该是各只票风险的"某种加权和"。
**最朴素的猜测** (错的):

```
组合方差 = Σ wᵢ² × σᵢ²    (只考虑每只票自己的方差)
```

**实际正确公式**:
$$
\sigma_p^2 = \mathbf{w}^T \Sigma \mathbf{w} = \sum_{i,j} w_i w_j \Sigma_{ij}
$$

注意有 **i ≠ j** 的项 (非对角项), 也就是**两只票之间的相关性会影响组合风险**。

为什么这是组合优化的灵魂?

```
情景 1: 两只票相关性 +0.9 (同涨同跌)
        各投一半: 组合波动 ≈ 单只票波动 (没起到分散作用)

情景 2: 两只票相关性 -0.9 (反向)
        各投一半: 组合波动 ≈ 0  (完美对冲!)

情景 3: 两只票相关性 0 (无关)
        各投一半: 组合波动 ≈ 单只票波动 / √2  (经典 "分散降险")
```

**协方差矩阵的非对角项捕捉的就是这个"分散效应"**。 等权策略忽略它, 组合优化使用它。

## 三、组合波动率公式直觉化

记住这个三步走:

```
1. 选权重 w (一个向量, 长度 = 股票数)
2. 计算 Σ × w (得到一个向量, 表示每只票对组合方差的"边际贡献")
3. 计算 w' × (Σw) (一个标量, 就是组合方差)
       └─ 再开方就是组合波动率
```

代码:
```python
import numpy as np
w = np.array([0.3, 0.4, 0.3])
Sigma = np.array([[0.04, 0.02, 0.001],
                  [0.02, 0.09, 0.005],
                  [0.001, 0.005, 0.16]])

port_var = w @ Sigma @ w        # 标量
port_vol = np.sqrt(port_var)    # 日波动率
print(port_vol)
```

## 四、怎么估计 Σ

实战中, 你不会"凭空" 知道协方差矩阵, 必须从历史数据估。

最简单做法 (本框架默认):
```python
import pandas as pd
returns = prices.pct_change().dropna()
Sigma = returns.cov()  # 一行代码搞定
```

但**样本协方差矩阵的问题**:

### 1. 噪声大
你用 60 天历史估 N=30 只票的 Σ, Σ 里有 30×31/2 = 465 个数, 平均一个数对应 60/465 ≈ **0.13 个样本**。
**严重欠采样**, 估计结果噪声极大。

### 2. 维度高时不可逆
当 N (股票数) > T (观测期), Σ 矩阵**奇异不可逆**, 后续优化跑不动。

### 3. 历史 ≠ 未来
2019 年的 Σ 跟 2020 年 (新冠) 的 Σ 完全不同。 用历史估"未来一周"的协方差, 永远是偏差。

## 五、协方差矩阵的"工程升级"

业界为解决上述问题, 发展出几种改进方法 (本框架不实现, 但你要知道):

### 1. 收缩估计 (Shrinkage, Ledoit-Wolf 2003)
把"样本 Σ" 和"先验 Σ" (如对角阵) 加权平均:
$$
\hat{\Sigma} = (1 - \alpha) \Sigma_{sample} + \alpha \Sigma_{prior}
$$
- α 大: 偏向先验 (稳定但有偏)
- α 小: 偏向样本 (无偏但噪声大)
- Ledoit-Wolf 给了一个最优 α 的解析公式

实务中**收缩估计几乎总是比纯样本好**, 是机构标配。

### 2. 因子模型 (Barra / Axioma)
把 Σ 拆成"几个共同因子 + 残差":
$$
\Sigma = B F B^T + D
$$
- B: 资产对因子的暴露 (size × value × momentum × ...)
- F: 因子的协方差 (低维)
- D: 残差方差 (对角)

这样 N=1000 只票的 Σ 只用估 K=20 个因子的 F, 噪声大幅降低。

### 3. 滚动 / 加权
用过去 60 天 (不是全部历史), 或者用 EWM (近期权重高), 让 Σ 跟得上市场变化。

## 六、本框架对 Σ 的处理

为了小白友好, 本框架对 Σ 的处理很简单:

```python
# src/portfolio/optimizer.py
def _safe_cov(cov, jitter=1e-8):
    """给协方差矩阵对角加微小扰动, 避免数值奇异 (实战标配)."""
    n = cov.shape[0]
    return cov + jitter * np.eye(n)
```

只做一件事: **对角加 jitter**, 保证可逆。 不做收缩、不做因子模型, 这些都是后续优化方向。

OptimizedStrategy 在调仓时用最近 `lookback_days` (默认 60) 天的收益估 Σ, 然后送给优化器。

## 七、协方差矩阵在组合优化里的角色

后面 4 篇方法, 大致都依赖 Σ:

| 方法 | 需要 Σ? | 需要 μ (期望收益)? |
|---|---|---|
| 等权 | 否 (但摘要用) | 否 |
| 反波动率 | 是 (只用对角线) | 否 |
| 风险平价 | 是 (全用) | 否 |
| 最小方差 | 是 (全用) | 否 |
| 均值方差 | 是 (全用) | **是** |
| 最大夏普 | 是 (全用) | **是** |

**重点**: 只需要 Σ 的方法 (前 4 个) 在实战里**更稳健**, 因为 Σ 比 μ 好估太多。
后两个需要 μ 的方法理论上更优, 实战极易因 μ 估不准而暴雷。

## 八、小结

- 协方差矩阵 Σ 是组合优化的核心输入
- 组合方差 = w' Σ w, 关键是非对角项 (相关性) 捕捉分散效应
- 实战中 Σ 极难估准 (噪声大、维度高、非平稳)
- 业界用 shrinkage / 因子模型改善, 本框架用最简化的 jitter 兜底
- **只需要 Σ 的方法在实战更稳健**, 这是后面"为什么 min_var/risk_parity 更主流" 的原因
