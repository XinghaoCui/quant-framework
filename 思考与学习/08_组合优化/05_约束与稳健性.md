# 约束与稳健性

> 把理论模型搬到实战, 必踩的坑。
> 这一篇帮你提前知道, 别等亏了再后悔。

## 一、为什么需要约束

数学上"最优" 的权重经常实战不可执行:
- 算出来 30% 押在一只票上 → 风控不让
- 算出来某只票做空 80% → A 股做不了空
- 算出来跨行业暴露 60% 在金融 → 行业风险敞口超限

实战中, 组合优化几乎从不"无约束求解", 总要加各种约束。

## 二、最常见的 6 类约束

### 1. 权重和约束 (Budget Constraint)
$$
\sum w_i = 1
$$
全部资金都投出去 (不留现金), 本框架默认这个。

变体:
- $\sum w_i \leq 1$: 允许留现金 (保守)
- $\sum w_i = 1, w_i \geq 0$: 不做空 (A 股标配)
- $\sum |w_i| \leq 2$: 杠杆 2 倍

### 2. 做空约束 (No-Short Constraint)
$$
w_i \geq 0
$$
A 股个人投资者必须用这个。 美股可以放开但要谨慎。
本框架默认 `allow_short=False`, 想做空就显式打开。

### 3. 单票权重上限 (Concentration Limit)
$$
w_i \leq w_{\max}
$$
防止"全押一只票"。 经验值:
- 公募基金: 单票不超过 10% (法规)
- 机构组合: 5% 上限常见
- 你做研究: 至少别超过 30%

本框架支持: `min_variance_weights(returns, max_weight=0.10)`

### 4. 行业 / 风格暴露约束
$$
|\sum_{i \in \text{sector}} w_i - b_{sector}| \leq \delta
$$
让金融行业的总权重不超过基准的 ±5%。 这叫"风险中性化", 机构实战常用。 本框架未实现 (需要行业归类数据)。

### 5. 换手率约束
$$
\sum |w_i^{new} - w_i^{old}| \leq T
$$
本次调仓的换手不超过 T。 防止"理论最优, 但每次全换" 导致交易成本爆炸。

### 6. 跟踪误差约束
$$
(w - w_{bench})' \Sigma (w - w_{bench}) \leq T E^2
$$
组合相对基准的偏离方差不能超过 TE²。 增强指数策略标配。

## 三、稳健性: μ 和 Σ 估不准怎么办

组合优化是个"垃圾进, 垃圾出" 的过程。 μ 和 Σ 估得差, 优化算出的权重也差。

### 1. 协方差矩阵 shrinkage (Ledoit-Wolf)
最实用的一个改进:
$$
\hat{\Sigma} = (1 - \alpha) \Sigma_{sample} + \alpha (\text{trace}(\Sigma_{sample}) / N) \cdot I
$$
α 越大越偏向"对角单位阵" 这个先验。 Ledoit-Wolf 给了 α 的最优解析解。

sklearn 里有:
```python
from sklearn.covariance import LedoitWolf
lw = LedoitWolf().fit(returns)
Sigma_shrunk = lw.covariance_
```

本框架未集成, 但你可以自己调 sklearn 然后丢给优化器:
```python
from src.portfolio import min_variance_weights
res = min_variance_weights(pd.DataFrame(Sigma_shrunk, index=returns.columns, columns=returns.columns))
```

### 2. 期望收益 shrinkage
最简单做法: 把所有 μ 收缩到全市场均值:
$$
\hat{\mu}_i = (1-\alpha) \bar{\mu}_i + \alpha \bar{\mu}_{market}
$$

更高级: Black-Litterman 模型 (见上一篇)。

### 3. 滚动窗口而非全样本
不要用 5 年全部历史估 Σ (旧规律污染), 用 1 年滚动窗口更贴合当前市场。
本框架 `OptimizedStrategy(lookback_days=60)` 默认用 60 天滚动。

### 4. 加 jitter 防奇异
$$
\hat{\Sigma} = \Sigma + \epsilon I
$$
小幅扰动保数值稳定。 本框架默认 jitter=1e-8。

## 四、实战的"组合优化检查清单"

每次调用优化器后, 不要直接相信结果, 先过这 5 个检查:

```python
res = min_variance_weights(returns, max_weight=0.10)
w = res.weights

# 检查 1: 优化器是否收敛
assert res.success, "优化器没收敛, 检查 Σ 是否奇异"

# 检查 2: 权重和近似为 1
assert abs(w.sum() - 1.0) < 1e-4

# 检查 3: 没有极端权重
assert w.max() < 0.5, f"最大权重 {w.max():.2%} 过高"

# 检查 4: 集中度合理
hhi = (w ** 2).sum()
assert hhi < 0.3, f"HHI = {hhi:.2f}, 太集中"

# 检查 5: 跟基准 (等权) 对比波动
eq_vol = (np.ones(len(w)) / len(w)) @ returns.cov().values @ (np.ones(len(w)) / len(w))
assert res.expected_vol ** 2 <= eq_vol * 1.5, "优化后比等权还波动大?"
```

## 五、为什么"复杂方法常常输给等权"

学术界有一个著名实证 (DeMiguel, Garlappi, Uppal 2009):

> 14 个最先进的组合优化方法, 在样本外几乎都打不过 1/N 等权。

为什么?
- 这些方法都依赖 μ 或 Σ 估计
- 估计误差加上样本外的 régime 切换, 理论"最优"就变成了实战"过拟合"
- 等权是个极强的贝叶斯先验 (所有票期望收益和风险一样), 不需要任何估计

所以别迷信"复杂方法肯定更好"。 永远跟等权对比, 没明显赢就别用复杂方法。

这也是为什么 `examples/30` 里我们做了详细对比, 你能看到 min_variance 只比等权略稳一点, 不是数量级的提升。

## 六、组合优化在量化生态里的位置

新手常把"组合优化" 当成"赚钱秘籍", 这是错的。 它的真实价值:

| 模块 | 贡献 |
|---|---|
| 因子研究 (07 章) | 决定收益从哪来 (alpha 来源) |
| 组合优化 (本章) | 决定风险/收益怎么平衡 (不增加 alpha, 但提高夏普) |
| 风险管理 | 决定最大能亏多少 (止损、风控) |
| 执行优化 | 决定实际成交价格多少 (减少滑点) |

> 组合优化是"放大器", 不是"创造者"。 因子没 alpha 时, 优化得再精也救不了。

## 七、小结

- 实战必加约束: 权重和、不做空、单票上限、行业暴露、换手率、跟踪误差
- μ/Σ 估不准是组合优化的最大敌人, 用 shrinkage / 滚动窗口 / Robust 应对
- 每次优化后过 5 项检查清单, 别直接相信结果
- 复杂方法在样本外常常输给等权, 永远做对比
- 组合优化是"放大器", 不是"创造者"
- 它能让有 alpha 的策略夏普从 1.0 涨到 1.3, 但不能让无 alpha 策略赚钱
