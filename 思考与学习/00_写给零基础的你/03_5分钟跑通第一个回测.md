# 5 分钟跑通第一个回测

> 一行命令, 看到你人生第一张回测报告。
> 这一篇的核心不是"理解", 而是**让你先有成就感**。

## 心理建设

这一篇你**不需要看懂任何代码**。
就当魔法师念个咒, 看东西出现就行。原理留到下一篇 04 慢慢讲。

> "**先看到结果, 再回头理解为什么。**" —— 学新东西最快的方式。

## 一、确认环境已准备好

跟着 [02 环境准备](./02_环境准备.md) 走完了的话, 现在你应该:
- 在 `quant-framework` 目录里
- 命令行最前面有 `(.venv)` 标识

没有的话, 回去补 02。

## 二、敲这一行命令

```bash
python examples/00_模拟数据_全策略对比.py
```

> Windows 上如果出现中文乱码, 改成:
> ```bash
> set PYTHONIOENCODING=utf-8 && python examples/00_模拟数据_全策略对比.py
> ```
> Mac/Linux 通常不需要。

## 四、刚才发生了什么（粗略版）

```
1. 程序生成了 20 只虚构股票, 7 年的日线价格（"模拟数据"）
2. 用三个不同的策略各跑了一遍这 7 年:
   - SmaCross  (双均线): 短均线穿长均线时买入
   - Momentum (动量):   过去涨得好的继续买
   - ValueFactor (价值): 过去跌得多的反向买入
3. 对每个策略, 算出 11 个指标（夏普、回撤等等）
4. 最后给一张三策略横比表
5. 顺手把净值曲线图保存到 reports/ 目录里
```

下一篇 [04 看懂第一张回测报告](./04_看懂第一张回测报告.md) 会把那些指标逐条讲透。

## 五、去看看保存的图

打开 `reports/` 目录, 你会看到三个文件夹（一个策略一个）。
点进去随便一个, 会看到:
- `equity_curve.png` — 净值曲线图（你的钱怎么涨的）
- `drawdown.png` — 回撤曲线图（你最多亏过多少）
- `metrics.csv` — 所有指标存成 csv

用图片查看器打开 PNG, 你会看到一条向右上方爬的曲线（净值）。
**那就是你跑出来的、第一条属于你的回测净值曲线。**

## 六、为什么不直接用真实数据

模拟数据有三个好处:

1. **不需要联网**: 真实数据要从 yfinance 或 akshare 抓, 受网速、限流、代理影响
2. **不会因为外部服务挂了你也卡住**: yfinance 时不时报 "Too Many Requests"
3. **跑一遍只要几秒**, 调试代码贼快

等你熟练了, 可以跑 `examples/01_双均线_美股.py` 看真实美股, 或者 `examples/03_价值因子_A股.py` 看真实 A 股。
**但第一次, 一定先跑 examples/00**, 因为它最不容易出错。

## 七、试着改一个参数（强烈推荐）

打开 `examples/00_模拟数据_全策略对比.py`, 找到这一行:
```python
SmaCrossStrategy(fast=20, slow=60, allow_short=False),
```

把 `fast=20, slow=60` 改成 `fast=5, slow=20`, 然后再跑一次。
观察:
- 累计收益变了多少?
- 夏普变高了还是变低了?
- 换手率变了多少?

你会**手指头先于大脑**地感受到"参数怎么影响策略表现"。
这种**实验感**比读十遍理论都管用。

## 八、跑通了, 但有没有看懂?

**没看懂太正常了。** 你刚刚:
- 看到了一堆数字, 但不知道夏普 1.69 算好算坏
- 看到了"累计收益 202%", 但不知道这是不是过拟合
- 看到了"最大回撤 -16%", 但不知道这意味着什么

这些都是下一篇的内容。

## 九、如果跑失败了

不要慌, 90% 的人会卡在这里某一步。常见错误:

| 报错 | 大概率原因 | 看哪里 |
|---|---|---|
| `ModuleNotFoundError: No module named 'pandas'` | 虚拟环境没激活 | [02 环境准备](./02_环境准备.md) |
| 中文乱码 (一堆问号) | 编码设置 | [06 FAQ](./06_常见报错与FAQ.md#中文乱码) |
| `python: command not found` | Python 未装 / PATH 未配 | [02 环境准备](./02_环境准备.md) |
| `FileNotFoundError: examples/00...` | 没在项目根目录 | `cd` 到 `quant-framework/` 再跑 |

更多坑见 [06 常见报错](./06_常见报错与FAQ.md)。
