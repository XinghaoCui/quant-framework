# 完全没编程基础怎么办

> 给真小白的最小编程清单。
> 你不需要"学会 Python", 只需要"看得懂示例代码 + 能改一两个参数"。

## 心理建设

**好消息**: 量化研究员的代码水平**远低于**软件工程师。 我们不写大型系统, 只写"分析脚本"。
**更好的消息**: 你只需要会用 Python + pandas 的几个常用操作, 就能读懂本仓库 80% 的代码。

> "**会写代码"≠"会量化"**。
> 见过很多 CS 大神写不出靠谱策略, 也见过很多金融背景的人用初级 Python 做出顶级研究。
> **量化拼的是金融思维 + 数据感觉, 不是代码水平。**

## 你需要会到什么程度

具体到能力:

### 必须会 (1 周搞定)
- [ ] **变量和数据类型**: `x = 10`, `name = "hello"`, `prices = [1, 2, 3]`
- [ ] **函数调用**: `print(x)`, `len(prices)`
- [ ] **if / for 循环**:
  ```python
  for x in prices:
      if x > 5:
          print(x)
  ```
- [ ] **导入库**: `import pandas as pd`
- [ ] **dict / list 基本操作**: `d["key"]`, `lst.append(x)`

### 应该会 (再 1 周)
- [ ] **pandas Series / DataFrame** 是什么, 怎么取一列
- [ ] **dataframe.loc[date]** 取某一行
- [ ] **df.mean(), df.std()** 这些聚合
- [ ] **df.shift(1)** 是把数据往后挪一行（量化里超常用）
- [ ] **df.pct_change()** 算收益率

### 不用立刻会 (用到再查)
- 装饰器 (`@property`, `@dataclass` 等)
- 类继承、`abstractmethod`
- type hint (`List[int]`, `Optional[X]`)
- async / multiprocessing
- generator / yield

> **看不懂的部分先跳过**, 量化代码的"骨架"很简单, 装饰器之类的细枝末节不影响你理解逻辑。

## Pandas 速通: 5 个最常用操作

本仓库的代码核心就是这 5 个。 弄懂它们, 80% 的代码看得懂。

### 1. DataFrame 是什么
```python
import pandas as pd

# 一个 DataFrame 就是一张表（像 Excel）:
df = pd.DataFrame({
    "AAPL": [100, 102, 105],
    "MSFT": [200, 201, 199],
}, index=["2024-01-01", "2024-01-02", "2024-01-03"])

#                    AAPL  MSFT
#  2024-01-01        100   200
#  2024-01-02        102   201
#  2024-01-03        105   199
```

本仓库里的 `prices` 就是这种"日期 × 股票"的宽表。

### 2. 取数据
```python
df["AAPL"]                  # 取一列 → Series
df.loc["2024-01-02"]        # 取一行 → Series
df.iloc[0]                  # 取第 0 行
df.iloc[:, 0]               # 取第 0 列
```

### 3. 算收益率
```python
returns = df.pct_change()
# 每个值变成 "比上一行涨了多少 %"
#                    AAPL   MSFT
#  2024-01-01        NaN    NaN
#  2024-01-02        0.02   0.005
#  2024-01-03        0.029  -0.010
```

### 4. shift = 把数据往后挪
```python
df.shift(1)
# 每一行变成"前一天的值"
# 量化里 95% 用它来防"未来函数":
#   昨天算出的信号, 今天才能用 → df.shift(1)
```

### 5. 聚合
```python
df.mean()         # 每列均值
df.std()          # 每列标准差
df.cumprod()      # 累乘（算累计净值要用）
df.corr()         # 列间相关系数
df.corrwith(other, axis=1)   # 按行算两个 df 的相关性（IC 就靠它）
```

会这 5 个, 本仓库的 `evaluation.py`、`metrics.py` 都能看懂。

## NumPy 速通: 不用学, 跟着用

NumPy 主要是给 Pandas 当后台的, 你一般直接看到的是:
- `np.sqrt(x)` — 开方（年化波动率要乘 √252）
- `np.log(x)` — 对数（算对数收益率）
- `np.nan` — 缺失值标记

就这些, 用到的时候照抄就行。

## sklearn 速通: 用到 06 章 ML 时再学

sklearn 的核心 API 就 4 行:
```python
from sklearn.linear_model import Ridge

model = Ridge(alpha=1.0)   # 初始化模型
model.fit(X_train, y_train)  # 训练
preds = model.predict(X_test)  # 预测
```

会这 4 行就能跑通本仓库 06 章所有 ML 示例。 等你真要调参时再深学。

## VS Code 几个超有用的快捷键

- `Ctrl+P`: 快速跳文件（你想打开 `evaluation.py`, 不用一层层点目录, `Ctrl+P` 输 "eval" 就行）
- `Ctrl+Shift+F`: 全局搜索（"哪里用到了 compute_ic?" 一搜就有）
- `F12`: 跳到定义（光标停在 `MultiFactorStrategy` 上按 F12 → 跳到它的源码）
- `Alt+Shift+F`: 自动格式化代码

学会这 4 个, 阅读效率翻倍。

## 不要做的事

- ❌ **不要先把《Python 入门》读完再来看本仓库**。 你会读到一半就忘了。 边用边学。
- ❌ **不要纠结"为什么不直接用 Java/C++"**。 量化用 Python 是行业事实, 学就完了。
- ❌ **不要害怕看英文文档**。 浏览器自带翻译, AI 可以解释, 没那么可怕。
- ❌ **不要把自己定位为"程序员"**。 你是**量化研究员**, 代码是工具不是目标。 写得能跑就行, 不必追求"优雅"。

## 如果你完全不想学编程怎么办

但你可以:
- 读懂报告、读懂研究方法 → 适合做**量化产品经理 / 销售 / 风控**
