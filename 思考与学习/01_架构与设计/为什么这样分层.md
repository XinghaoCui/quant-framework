# 为什么这样分层：数据 / 引擎 / 策略 / 因子

> 第一次来? 推荐先看 [00 章 / 写给零基础的你](../00_写给零基础的你/00_读我.md), 跑通示例再回来读架构原理。

## 一图概览

```
        ┌─────────────────────────────────────────┐
        │           用户脚本 (examples/)          │
        │  "我要用 XX 策略在 XX 标的上跑 XX 时段"   │
        └─────────────────────────────────────────┘
                          │
        ┌─────────────────┴─────────────────┐
        ▼                                   ▼
   ┌─────────┐    返回价格 DataFrame    ┌──────────┐
   │ 数据层  │ ────────────────────────▶│  策略    │
   │  data/  │                          │strategies│
   └─────────┘                          └──────────┘
   yfinance/akshare 统一接口              输出 weights
                                              │
                                              ▼
                                       ┌──────────┐
                                       │ 回测引擎 │
                                       │  engine/ │
                                       └──────────┘
                                       计算收益、扣成本、
                                       算所有指标
```

## 核心原则：每一层只做一件事

### 数据层 (data/)
这一层只做一件事：把任何数据源都包装成同一种 DataFrame。

```
index: 交易日 (DatetimeIndex)
columns: 标的代码 (str)
values: 后复权收盘价 (float)
```

为什么必须统一？因为如果不统一，每个策略都要写一遍"yfinance 字段是 `Adj Close`、akshare 是 `收盘`、复权方式不同"的代码，修一个 bug 要修十遍。

### 策略层 (strategies/)
这一层只做一件事：根据价格输出"目标仓位矩阵"。

```python
class Strategy:
    def generate_weights(self, prices: pd.DataFrame) -> pd.DataFrame:
        ...
```

策略**不应该**关心：
- 数据是怎么来的（不要在策略里调 yfinance）
- 交易成本是多少（那是引擎的事）
- 怎么画图（那是 utils 的事）
- 是不是有未来函数（引擎会 shift）

策略**应该**关心：
- "在 t 日盘后，我想让组合长什么样"

### 引擎层 (engine/)
这一层只做一件事：把策略输出的权重 + 价格数据，变成收益曲线和指标。

引擎处理所有"会让人犯错"的细节：
- shift(1) 防止未来函数
- 换手率计算
- 扣减交易成本
- 计算夏普、回撤等指标

### 因子层 (factors/)
这一层只做一件事：把价格变成"打分"。

因子是策略的"原材料"。比如动量因子 = 过去 12 个月收益率，
策略可以基于这个因子写排序选股逻辑。

## 这样分层换来的好处

### 1. 换数据源不影响策略
今天用 yfinance，明天换成 Bloomberg，策略代码一行不用改——
因为 `prices` 这个 DataFrame 的接口是固定的。

### 2. 换策略不影响引擎
我可以一周写 10 个策略，引擎一行不用动。
新策略只需要继承 `Strategy` 基类、实现 `generate_weights`。

### 3. 换回测口径不影响策略
想加新的指标（如 IR、最大连亏天数）？改 `engine/metrics.py`，
所有策略立刻享受到，不需要改一行策略代码。

### 4. 防止常见错误（未来函数）
所有策略默认走引擎的 `shift(1)`，没办法忘记。
这是回测的头号大坑，新手 90% 的"惊人收益"都源于此。

## 反例：错误的分层

很多新手代码长这样：

```python
def my_strategy():
    prices = yf.download("AAPL")       # 1. 数据
    prices = prices.dropna()            # 2. 清洗
    sma20 = prices["Close"].rolling(20).mean()  # 3. 因子
    signal = sma20 > sma20.shift(1)     # 4. 信号
    pnl = prices["Close"].pct_change() * signal  # 5. 没 shift！
    print(pnl.cumsum())                 # 6. 输出
```

问题：
- 数据/因子/信号/回测全揉一起，复用不了
- 第 5 步没 shift，是经典未来函数：你今天用今天的信号去赚今天的钱
- 没有交易成本，没有指标，没有报告

我们的分层把这 6 步分到了 4 个模块，每个模块都很短，但合起来就是个完整系统。

## 量化求职的关键

面试时，能讲清楚"为什么这么分层"比"我跑出了多高的夏普"重要得多。
分层好的代码 = 知道工业界长什么样 = 信号好。
