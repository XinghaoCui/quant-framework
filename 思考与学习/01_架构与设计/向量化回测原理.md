# 向量化回测原理

## 问题：怎么算回测收益最快？

假设有 500 只票、10 年日线 ≈ 2500 个交易日。
一个策略每天对每只票决定一个权重，求最终的策略累计收益。

### 朴素写法（事件驱动）

```python
cash = 100_000
positions = {sym: 0 for sym in symbols}
for date in dates:
    for sym in symbols:
        new_pos = strategy.target_position(sym, date)
        delta = new_pos - positions[sym]
        cash -= delta * prices.loc[date, sym]
        positions[sym] = new_pos
```

10 年 × 500 票 = 125 万次循环。Python 单线程跑要 30 秒到几分钟。

### 向量化写法

```python
asset_returns = prices.pct_change()
gross_returns = (weights.shift(1) * asset_returns).sum(axis=1)
```

两行。在 numpy 底层 C 代码里执行，10 年 × 500 票只要 几十毫秒。

## 数学上为什么对？

组合的日收益率 = 各资产收益的加权和：

$$
r_p(t) = \sum_i w_i(t-1) \cdot r_i(t)
$$

注意 $w_i$ 用的是 $t-1$ 时刻的权重（昨天盘后决定的仓位，今天才能持有产生收益）。这就是 `shift(1)` 的来源。

矩阵化：

- 权重矩阵 W：行=日期，列=股票
- 收益率矩阵 R：行=日期，列=股票
- 组合日收益 `r_p = (W.shift(1) * R).sum(axis=1)`：是一个时间序列

整个回测就是一次矩阵元素乘法 + 一次按行求和，所以快。

## 防未来函数（look-ahead bias）

> "未来函数"是指**在 t 日的决策用了 t 日及以后才能知道的信息**。

例子：
```python
weights[date] = (close[date] > sma20[date])  # 错
```
这里 `close[date]` 是 `date` 日的收盘价，但要在收盘后才知道。
如果你用这个信号在 `date` 日**当天**交易，你赚的是不存在的钱。

正确做法：
```python
weights[date] = (close[date] > sma20[date])  # 信号在 date 日盘后产生
# 引擎自动 shift(1)：实际持仓 = weights.shift(1)
# 即 date+1 日才用上 date 日产生的信号
```

我们的引擎里这一步是**强制**的：

```python
positions = weights.shift(1).fillna(0.0)
```

策略作者根本没法绕过它。这是个"系统性防错"设计。

## 向量化能做什么、不能做什么

### 能做

- 横截面选股：每天对全市场打分排序，选前 N
- 时序信号：均线、动量、布林带
- 多因子合成：把多个因子矩阵加权
- 大规模因子测试：几十个因子 × 几千股票 × 几十年

### 不能做（向量化框架的局限）

- 限价单、止损止盈：这些是"价格触发"的，需要逐 bar 撮合
- 订单簿、滑点模型：需要 tick 数据和事件驱动
- 复杂资金管理：如风险预算、动态杠杆调整、margin call

这些场景应该用 **backtrader** / **zipline** 这种事件驱动引擎。
我们这个框架定位是"研究级回测"，验证 alpha 信号本身是否有效，
没必要为了一个不存在的限价单逻辑把代码搞复杂。

## 交易成本模型

实盘的总成本 = 佣金 + 印花税 + 滑点 + 冲击成本：

- 佣金：券商收费，A 股万 2.5 单边，美股零佣金（但有 SEC fee）
- 印花税：A 股卖出收千 1，买入不收
- 滑点：你想 100 元买，实际成交 100.05，差 5 分就是 5 bps 滑点
- 冲击成本：大单会推动价格，进出场都会损失

在向量化框架里，所有成本简化为：

```python
cost = turnover * (commission + slippage)
```

其中 `turnover = |w_t - w_{t-1}|` 求和。这个简化在"换手率不太高"的中低频策略里足够精确。高频策略不适用。

## 默认成本设置

- 美股：`commission=0.0003, slippage=0.0001` 总 4 bps（接近真实零佣金 + 中等流动性）
- A 股：`commission=0.0008, slippage=0.0002` 总 10 bps（含印花税）

如果策略在这个成本下还赚钱，才有意义。很多论文里的"夏普 3.0"加上 5 bps 成本后就变成 0.5。
