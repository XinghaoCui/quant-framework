## 一、为什么"技术 + 基本面" 组合特别好

这件事数学上有保证。

08 章组合优化讲过: 几个低相关因子合成, IR ≈ 单因子 IR × √k 倍提升。

技术因子 vs 基本面因子相关性极低 (经验上 < 0.2), 远低于"同类因子之间" 的相关性 (动量 vs 反转 corr 高达 -0.5+, 价值 vs 质量 corr 也常达 0.3+)。

所以技术 + 基本面合成几乎是免费的午餐:
- 两边都不放弃任何 alpha
- 相关性低 → 综合 IR 提升明显
- 不同 régime 下表现互补 (技术因子在动荡市好, 基本面在长期价值修复市好)

## 二、完整代码路线

```python
import pandas as pd
from src.data import generate_synthetic_prices
from src.engine import Backtester
from src.factors import (
    momentum, reversal, volatility,
    roe_factor, earnings_yield, profit_growth_factor,
)
from src.strategies import MultiFactorStrategy, OptimizedStrategy
from src.strategies.multi_factor import cross_section_zscore

# 1. 数据
prices = generate_synthetic_prices(
    [f"STK{i:02d}" for i in range(30)],
    "2020-01-01", "2024-12-31", seed=42,
)

# 2. 全因子库 (技术 + 基本面)
all_factors = {
    # 技术因子家族
    "momentum":  momentum(prices, lookback=252, skip=21),
    "reversal":  reversal(prices, window=21),
    "lowvol":   -volatility(prices, window=60),
    # 基本面因子家族
    "ROE":           roe_factor(prices, use_synthetic=True),
    "EP":            earnings_yield(prices, use_synthetic=True),
    "profit_growth": profit_growth_factor(prices, use_synthetic=True),
}

# 3a. 方案 A: 直接 MultiFactorStrategy (ic_weighted 自动找方向)
strat = MultiFactorStrategy(
    factors=all_factors,
    combine="ic_weighted",
    ic_lookback=60,
    ic_horizon=5,
    top_pct=0.3,
    rebalance_days=21,
)
weights = strat.generate_weights(prices)

# 3b. 方案 B: 先 zscore 合成, 再丢给 OptimizedStrategy 做风险分仓
zs = [cross_section_zscore(f.reindex_like(prices)) for f in all_factors.values()]
composite = sum(zs) / len(zs)
opt_strat = OptimizedStrategy(
    scores=composite,
    optimizer="min_variance",
    top_pct=0.3,
    rebalance_days=21,
    lookback_days=60,
    max_weight=0.4,
)
opt_weights = opt_strat.generate_weights(prices)

# 4. 回测
bt = Backtester(commission=0.0003, slippage=0.0001)
result_A = bt.run(prices, weights)
result_B = bt.run(prices, opt_weights)

print("方案 A (ic_weighted):", result_A.metrics)
print("方案 B (zscore + min_var):", result_B.metrics)
```

## 五、再进一步: 你可以做的扩展

### 扩展 1: 加入行业归类 (中性化)
真实研究里, 因子值必须做"行业中性化": 同一行业内排序, 不跨行业。
需要拉行业归类数据 (akshare 有 `stock_sector_spot`), 然后改 `quantile_labels` 函数。

### 扩展 2: 加入 ETF / 指数作为基准
对比策略相对沪深 300 / 中证 500 的超额收益, 比绝对收益更有意义。

### 扩展 3: 接 Black-Litterman 把 ML 预测整合
06 章 ML 模型预测的收益, 可以作为 "投资者观点" 喂给 BL 模型, 然后用 BL 修正后的 μ 跑 mean_variance_weights。 这是机构标准做法。

### 扩展 4: 风险归因 / Brinson 分解 (10 章候选)
把组合收益拆成 "市场 / 风格 / 选股" 三部分, 知道 alpha 来自哪里。

### 扩展 5: 实盘对接
本框架是回测框架, 实盘要接券商 API:
- A 股: vn.py / QMT / Ptrade
- 美股: Interactive Brokers / Alpaca / Robinhood
- 港股: Futu API

实盘对接是另一个工程方向, 比研究更难: 涉及订单生命周期、撮合时序、风控规则、合规等。

## 六、跑 examples/43 的真实输出

最后看一遍这个示例的真实输出, 体会"全栈流水线" 的最终成果:

```
===== 策略对照表 =====
                        年化收益    夏普比率    最大回撤  Calmar比率  平均换手率(单边)
A_tech_only_equal     0.0923  0.9937 -0.1415    0.6523     0.0230
B_fund_only_equal    -0.0238 -0.2127 -0.3054   -0.0779     0.0066
C_combo_equal        -0.0242 -0.2168 -0.3100   -0.0781     0.0168
D_combo_ic_weighted   0.1821  1.9807 -0.0869    2.0955     0.0142
benchmark_equal_hold  0.0947  1.6566 -0.0639    1.4815     0.0004
```

观察:
- A (纯技术): 夏普 0.99, 还行但跑不过基准
- B (纯基本面): 夏普 -0.21, 模拟数据上基本面因子方向不顺
- C (等权合成): 夏普 -0.22, 被 B 拖累
- D (ic_weighted): 夏普 1.98, 大幅超越基准 ← 综合方案

D 的核心机制:
- ic_weighted 自动检测每个因子的近期 IC
- 给正 IC 因子正权重, 给负 IC 因子负权重 (自动反向用)
- 综合分既利用了技术因子, 又利用了反向的基本面因子
- 结果不同于任何单因子, 是几个因子组合出来的新 alpha

这就是多因子合成有意思的地方, 也是 ic_weighted 比 equal 更鲁棒的原因。
