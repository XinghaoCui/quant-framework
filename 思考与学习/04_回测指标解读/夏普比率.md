# 夏普比率（Sharpe Ratio）

> 想先看"零术语 + 对照真实输出"的指标解读? 看 [00 章 / 04 看懂第一张回测报告](../00_写给零基础的你/04_看懂第一张回测报告.md), 那一篇把夏普讲得更直观。本篇深入数学定义和陷阱。

## 一句话定义

> 单位风险（波动率）能带来多少超额收益。

## 数学定义

$$
\text{Sharpe} = \frac{E[R_p] - R_f}{\sigma_p}
$$

- $R_p$：策略收益率
- $R_f$：无风险利率
- $\sigma_p$：策略收益率的标准差

代码实现（年化）：
```python
def sharpe_ratio(returns, risk_free=0.0):
    excess = returns - risk_free / 252
    return excess.mean() / excess.std() * np.sqrt(252)
```

## 年化方式

如果用日收益：
$$
\text{Sharpe}_{annual} = \frac{\bar{r}_{daily}}{\sigma_{daily}} \cdot \sqrt{252}
$$

为什么是 √252？参考 [收益率与对数收益率](../02_核心术语/收益率与对数收益率.md) 的"年化波动率为什么是 ×√252"一节。

## 经验判读

| 夏普范围 | 判断 |
|---|---|
| < 0 | 策略亏钱 |
| 0 ~ 0.5 | 还不如买债 |
| 0.5 ~ 1.0 | 一般，市场基准（如标普长期 ~0.5） |
| 1.0 ~ 1.5 | 可投资 |
| 1.5 ~ 2.0 | 优秀 |
| 2.0 ~ 3.0 | 顶级（私募策略级别） |
| > 3.0 | **警惕过拟合** |
| > 5.0 | 几乎可以确定有 bug（未来函数、survivorship） |

实盘中：
- 顶级公募基金长期 Sharpe ~1.0
- 顶级对冲基金长期 Sharpe ~1.5-2.0
- Renaissance Medallion 长期 Sharpe ~6（但这是高频、有海量私有数据、不对外）

## 三个常见陷阱

### 1. 用日 Sharpe 直接对比月 Sharpe

如果不年化，日 Sharpe 和月 Sharpe 没有可比性。
任何报告里的 Sharpe **都应该是年化的**。

### 2. 没扣交易成本

毛 Sharpe 1.8 听起来不错，扣成本后可能只剩 0.4。
评估策略一定要看**净 Sharpe**。

### 3. 用错样本

- 只用牛市数据：Sharpe 虚高
- 包含 2008 / 2020 等极端时段：Sharpe 真实
- 至少要覆盖 1 个完整经济周期（10 年+）

## Sharpe 的局限

### 1. 假设收益正态分布

但实际收益有**肥尾**和**偏态**：
- 卖期权策略：日常稳定赚小钱，偶尔大亏（左偏）→ Sharpe 看起来很高，但崩盘风险大
- 趋势策略：经常小亏，偶尔大赚（右偏）→ Sharpe 看起来低，但实际不错

所以高 Sharpe 不等于策略好，**还要看回撤、最大单日亏损、肥尾**。

### 2. 不区分上行/下行波动

价格涨 5% 和跌 5% 都贡献"风险"，但前者是好事。
**Sortino 比率** 解决这个问题（只算下行波动）。

### 3. 对操纵很敏感

平滑收益（如月度结算的私募）会人为压低波动率，虚高 Sharpe。
学术界用 Bias-Adjusted Sharpe 来纠正。

## 跟其他比率的关系

| 比率 | 分子 | 分母 | 用途 |
|---|---|---|---|
| **Sharpe** | 超额收益 | 总波动 | 通用 |
| **Sortino** | 超额收益 | 下行波动 | 关注下行风险 |
| **Calmar** | 年化收益 | 最大回撤 | 关注极端损失 |
| **信息比率 IR** | 相对基准超额 | 跟踪误差 | 主动管理评估 |
| **Treynor** | 超额收益 | β | 系统性风险溢价 |

实务中常常**同时报告 Sharpe + Sortino + Calmar**，互相印证。

## 在本框架里

`src/engine/metrics.py` 的实现：

```python
def sharpe_ratio(returns: pd.Series, risk_free: float = 0.0) -> float:
    if returns.std(ddof=1) == 0 or len(returns) < 2:
        return 0.0
    excess = returns - risk_free / TRADING_DAYS
    return float(excess.mean() / excess.std(ddof=1) * np.sqrt(TRADING_DAYS))
```

注意细节：
- `ddof=1`：样本标准差（n-1 分母），不是总体（n）
- `TRADING_DAYS = 252`：固定常数，A 股美股都用 252（学术界惯例）
- 默认 `risk_free=0.0`：方便对比，需要时可传入实际值（如 4% / 252）
