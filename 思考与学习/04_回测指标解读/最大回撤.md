# 最大回撤（Maximum Drawdown, MDD）

## 一句话定义

> 从历史高点到后续低点的最大跌幅。
> 衡量"持有这个策略最难受的那一刻有多难受"。

## 数学定义

设净值序列 $N_t$，定义历史高点：

$$
\text{Peak}_t = \max_{s \le t} N_s
$$

回撤序列：

$$
DD_t = \frac{N_t}{\text{Peak}_t} - 1 \quad (\le 0)
$$

最大回撤：

$$
MDD = \min_t DD_t
$$

## 代码

```python
def max_drawdown(returns):
    net = (1 + returns).cumprod()
    peak = net.cummax()
    drawdown = net / peak - 1
    return drawdown.min()
```

## 为什么不是"最大跌幅"

很多新手算成"最高价 → 最低价"：
```python
mdd_wrong = (net.min() - net.max()) / net.max()
```

这是错的！因为最高点可能在最低点**之后**。
正确的回撤要求最高点在前、最低点在后。

`cummax` 强制"截止当前的历史最高"，保证时间顺序。

## 经验判读

| MDD | 判断 |
|---|---|
| 0 ~ -10% | 极稳健（低波动策略） |
| -10% ~ -20% | 稳健 |
| -20% ~ -30% | 一般（接近股市基准） |
| -30% ~ -50% | 高风险（典型股票策略） |
| < -50% | 极高风险（杠杆 / 集中持仓） |

参考：
- 标普 500 历史最大回撤（2008）约 -55%
- 纳指（2000-2002）约 -78%
- 比特币（2018，2022）多次 -80%+

## 为什么回撤 -50% 很可怕

跌 50% 后，需要 +100% 才能回本：

| 跌 | 需要涨回 |
|---|---|
| -10% | +11% |
| -20% | +25% |
| -30% | +43% |
| -50% | +100% |
| -70% | +233% |
| -90% | +900% |

**回撤和恢复是不对称的**。这就是为什么风险管理强调控制回撤。

## 回撤的其他维度

### 1. 回撤持续时间（Drawdown Duration）

从高点跌下来到重新创新高用了多久。
有些策略 MDD 不大但持续 3 年没创新高，对持有人是煎熬。

### 2. 平均回撤（Average Drawdown）

所有回撤期的均值。比 MDD 稳健，但少见报告。

### 3. 水下时间（Time Underwater）

不在历史新高的总时间占比。
长期策略通常 60-80% 的时间在水下。

## Calmar 比率

$$
\text{Calmar} = \frac{\text{年化收益}}{|MDD|}
$$

衡量"收益相对最痛点的性价比"。

| Calmar | 判断 |
|---|---|
| < 0.5 | 一般 |
| 0.5 ~ 1.0 | 可接受 |
| 1.0 ~ 2.0 | 优秀 |
| > 2.0 | 顶级 |

经验：长期策略 Calmar > 1 已经很好。
Sharpe 高但回撤大的策略，Calmar 会"修正"它的吸引力。

## 实战意义

### 风险预算

如果你能承受最大 -20% 回撤：
- 杠杆 1x，策略本身 MDD -20%
- 杠杆 0.5x，策略 MDD -40%（×0.5 = -20%）
- 杠杆 2x，策略 MDD -10%（×2 = -20%）

回撤是**线性可缩放**的（杠杆 k 倍 → 回撤 k 倍）。
所以"控制 MDD"等价于"控制杠杆"。

### 心理承受

> "回测 -30% MDD 看起来还能忍。
> 但你想象一下，账户从 100 万跌到 70 万、连续 6 个月没起色、每天看到亏损，你还能不能坚持？"

实盘 MDD 给你的痛苦远大于回测里的数字。
**保守估计**：实盘可承受 MDD ≈ 回测 MDD 的一半。

### 策略组合

低相关的策略组合可以**降低整体 MDD**：
- 策略 A：MDD -20%
- 策略 B：MDD -25%
- 50/50 组合：MDD 可能只有 -15%（相关性低的话）

这是"分散化"的核心价值——降低尾部风险，而不是提高收益。

## 在本框架里

`src/engine/metrics.py`:

```python
def max_drawdown(returns: pd.Series) -> float:
    if len(returns) == 0:
        return 0.0
    net = (1.0 + returns).cumprod()
    peak = net.cummax()
    drawdown = net / peak - 1.0
    return float(drawdown.min())
```
